# 융합설계 상태판

Last updated: 2026-06-19

이 폴더는 `claude`/`claude2` 지침을 `eh-universe-web`에 어떻게 흡수할지 정리한 내부 설계 원장입니다. 현재 공개 제품 기능 목록이 아닙니다.

## 읽는 법

| 표기 | 의미 |
|---|---|
| `wired` | 작성 당시 코드 seam이 있거나 일부 연결 가능성이 확인된 항목. 현재 제품 약속이나 전체 동작 보증이 아님. |
| `partial` | 일부 구조만 있음. UI/API/storage/test 중 빠진 축이 있음. |
| `design` | 설계 후보. 구현 착수 전 현재 제품 기준으로 재검토 필요. |
| `phantom` | 앱에 대응 코드가 없거나 미확인. 활성 기능처럼 설명 금지. |
| `[미매핑·확인필요]` | 에이전트 추정 또는 코드 근거 부족. 직접 grep, 타입, caller 확인 전에는 미구현으로 취급. |

## 현재 제품 기준

현재 공개 표면은 Loreguard Studio, Translation Studio, Docs, 가격/상태/법적 문서입니다.

구표면 이름이 이 폴더에 남아 있더라도 다음을 현재 제품으로 복구하지 않습니다:

- Code Studio
- Network
- Archive
- Codex
- Reports
- Reference
- Rulebook
- Tools

유효한 아이디어는 현행 용어로만 흡수합니다: 노아 인터뷰, 노아 제안, 과정기록, 참조 컨텍스트, 출고 패키지, 환경 설정, 히스토리.

## 특히 주의할 항목

- `_연극부_시스템.md`, `_backstage`, 리허설, 롤플레잉 계약은 설계 조상입니다. 현재 앱의 활성 기능이 아닙니다.
- 02/03의 `wired` 카운트는 “행별 완전 검증 완료”가 아닙니다. `_검증_표본.md`도 표본 5건 재검증입니다.
- 이 폴더를 근거로 UI 문구, 가격표, 외부 공지, 사용자 약속을 만들지 않습니다.

## 구현 전 게이트

새 기능으로 승격하려면 최소한 다음 증거가 필요합니다:

1. 현재 `Project` 데이터 구조와 충돌 없음
2. UI/API/storage/caller 중 필요한 축이 실제 코드에 연결됨
3. 관련 Jest 또는 Playwright 검증 통과
4. 공개 문구가 현행 Loreguard 용어와 일치
