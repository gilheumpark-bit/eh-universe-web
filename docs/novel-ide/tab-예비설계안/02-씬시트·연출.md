# 02 · 씬시트 / 연출 탭 (2탭) — 예비설계안 (대기)

> 근거: `_도구/04_씬시트_연출/`(18 모듈) · tunnel-yuju `씬시트/_씬시트_표준_양식.md`(직접 재검증) · `_정보_필요성_게이트.md`. 공통은 [`_공통-3기둥.md`](_공통-3기둥.md).

## 핵심: 씬시트 ≠ 연출 (2개 탭, 실물이 증명)
tunnel-yuju가 `씬시트/`와 `연출/`을 디렉토리로 분리. **씬시트 = "무엇이/왜"(8영역), 연출 = "어떻게 보이나"(컷 풀 11컬럼)**. 현재 `연출/`은 비어있음 — 씬시트가 먼저, 연출은 다운스트림. **커밋 단위 = 에피소드 1화**(씬 1개 아님; 화 내부는 5-Part / 감정 5영역으로 분절).

## A. 씬시트 탭 — `SceneSheet` (8영역, 활성)

```typescript
interface SceneSheet {
  episodeId: string;            // "ep_001"
  workId: string; title: string;
  mainStoryRef: string;         // ★ "메인 v8 §5/§7 {사건}" — 구성 탭 정합 링크
  // ★ [2026-06-06 M4 reconcile] lockHistory 폐기 — 씬시트는 기존 SceneDirectionDataV2(TaggedField, studio-types:159-178) 확장.
  // 필드 provenance = TaggedField<T>; 확정 이력 = OriginMetadata.editedBy[]. (신규 lockHistory 금지 — _공통 원칙2)
  refs: string[];   // refs = 캐릭/아이템/세계관 노드 (그래프 엣지)

  meta: {                        // §1 메타데이터
    chapterType: string[];       // 전투/협상/대치/각성/일상/위기돌파
    tempo: { speed; density; tension; emotion; action };  // 5요소 ★1~5
  };
  directionBrief: { intro; mid; end };          // §2 연출 개요(컷 풀은 연출 탭)
  characters: { role; name; refId?; motivation }[];  // §3 등장인물 & voice
  signaturePhrases: { character; line; context }[];  // §4 대사 톤
  emotionCurve: { tension; rising; crisis; climax; resolution; levels?:number[] };  // §5 감정 5영역
  rewardLoop: { frustration:string[]; dopamine:string[]; catharsis };  // §6 사이다/도파민/고구마
  hook: { types: HookType[]; lastThreeSentences: [string,string,string] };  // §7 절단신공
  infoGate: { necessityGate: InfoNecessityGate; forbiddenRules: string[]; silenceWindow? };  // §8

  advanced?: { aiTemperature?; beatRefs?: string[]; colorScript? };  // 13영역 정밀(접힘)
}
type HookType = "환경"|"메시지"|"등장"|"반전"|"시간점프"|"감각"|"정보누설"|"침묵";

// §8 핵심 — AI 숫자/날짜/풀네임 과잉 차단. 빈 배열 = 본문 생성 시 자동 차단(default-safe)
interface InfoNecessityGate {
  numbersNeeded: string[];   // "21: 어머니 검사 후 경과 년수"
  datesNeeded: string[]; agesNeeded: string[];
  timePreciseNeeded: string[];  // "0.4초: 형광등 박자 시그니처"
  fullNamesNeeded: string[];    // 그 외엔 대명사/약칭 강제
}
```

**13영역 vs 8영역**: instruction은 13영역 풀스펙, tunnel 실물은 8영역 다이어트. → **8영역 베이스 + 13영역 슬롯을 `advanced`로 접는** 구조(세계관 Tier1/2/3 점진공개와 동형).

**beat/긴장곡선/Peak-End/컬러스크립트/11컷 PV는 독립 탭이 아니라**:
- beat → `advanced.beatRefs` (Beat Bank motif ID 참조; data는 작가 영역)
- 긴장곡선 → `emotionCurve.levels[]`(micro) → 인덱스에서 meso/macro/epic 집계
- Peak/End → `emotionCurve.climax` + `aiTemperature.magicMomentFlag` + `hook.lastThreeSentences`(End)
- 컬러스크립트 → `advanced.colorScript` 또는 연출 `Shot.color`
- 11컷/60s PV → **연출 탭의 작품 단위 산출물**(외부 영업용)

## 채팅→폼 (씬시트)
8영역이 자연 폼 섹션. AI 채움 우선순위: §2 연출개요 → §5 감정곡선 → §6 리워드루프 → §7 훅. **§3 캐릭/§8 정보게이트는 작가 결정 영역** → AI는 후보 제안만(세계관/캐릭 노드에서 끌어옴). §1 메타는 회차 성격 선언 후 자동 보정.

## 정합검사 (씬시트 고유 — 공통 감사원에 더함)
1. **긴장 곡선 일관성** — 회차 `levels`를 인덱스 누적 → meso(5화 variety≥0.30)·macro(beat 매핑)·epic. 단조증가/plateau 경고.
2. **beat 누락** — foreshadow_status 변화에 대응 beat 미배치 / reusable=false motif 2회+.
3. **훅 중복** — 최근 5화 `hook.types` 교집합.
4. **정보게이트 위반** — 집필 후 본문 grep vs `necessityGate` 차집합(잉여 숫자/날짜/풀네임) → 작가 1회 묻기(차단/게이트추가/유지).
5. 밀도 ★5 연속 2화+ = 과부하 경고.
6. 씬-어미 정합(88셀): chapterType × 본문 종결어미 분포 (Confidence 0.55 — 작가 검증 영역).
7. (연출) `Shot.spoilerCheck`: 세계관 fact Restricted+ 자동 차단.

## 인덱스
**노드 = 화(에피소드)** — 세계관 WorldEntry와 동급. `refs[]`가 캐릭/아이템/세계관 노드로 엣지. 긴장곡선/Peak/컬러스크립트는 회차 노드 속성이지 별도 노드 아님 — 단 **인덱스 레벨에서 회차 가로질러 집계**(곡선 시각화·Peak 분포·훅 중복맵). 연출 노드 = 화 콘티(씬시트 화에 1:1) + 작품 PV 노드. tunnel 실물에 `_곡선_그래프/`·`_hook_log.md`·`_voice_dna_log.md`가 이미 보조 인덱스 프로토타입으로 존재.
