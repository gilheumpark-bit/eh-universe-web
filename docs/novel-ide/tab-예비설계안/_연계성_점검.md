# 기존 시스템 연계성 점검 (2026-06-03)

> 대상: 본 폴더 예비설계안 세트. 점검: 제안 설계 ↔ eh-universe-web 기존 Studio 실측 대조.
> **핵심 판정: 예비설계안이 greenfield(신규 스키마·신규 IndexedDB·신규 provenance·신규 relations·신규 coherence)로 그려졌으나, 기존 Studio가 그 개념의 ~70-80%를 이미 구현. 전면 재framing 필요 = "신규 생성"이 아니라 "기존에 어댑터 융합".**

## §2. 융합 원칙 (greenfield → fusion) — 4원칙

**원칙 1 — 단일 진실원 = 기존 `Project`.**
chat→form은 **새 입력 방식**일 뿐, 산출물은 **기존 구조**(Config 슬라이스: characters[]·worldSimData·episodeSceneSheets·sceneDirection)다. 별도 IndexedDB에 코어 데이터 사본 X(이중 진실원 = 데이터 파편화).

**원칙 2 — provenance/commit = 기존 M4 Origin Tag 재사용.**
chat→form이 채운 값 = `TaggedValue<T>{ value, meta:{origin:'ENGINE_DRAFT'} }`. 사람이 검토 수락 → `ENGINE_SUGGEST`, 직접 편집 → `USER`. **새 lockHistory 만들지 말 것** — `OriginMetadata.editedBy[]`가 곧 커밋 이력. "사람이 종료" = origin이 `USER`/`ENGINE_SUGGEST`로 확정되는 것.

**원칙 3 — 추가 메타만 sidecar (기존 id 키 참조).**
신규 모듈(worldgraph/charactergraph)은 **코어 데이터 복제 X**. 기존 entity id를 키로 *추가 메타*(tier_depth·spoiler·exposure·refs 그래프·coherence 결과)만 보유. 코어는 Project, 메타는 sidecar. madge: sidecar → studio-types **import 0**(타입은 구조적 복제 or 어댑터).

**원칙 4 — coherence = 기존 ProactiveSuggestion/QualityGate 확장.**
QA 감사원 A/B/C/D는 새 엔진이 아니라 `ProactiveSuggestion` 카테고리 확장 + 패널. world_inconsistency·character_drift·foreshadow_urgent가 이미 있음 — A 정합검사관이 이를 생성/강화. QualityGate 결과 스키마(`QualityGateResult`)에 감사원 list 부착.

## §3. 저장 접합부 (절대금지8 우회 — 실측 hook)
- 진입: `useProjectManager` / `usePrimaryWriter` / `useShadowProjectWriter` (hooks) — save-engine **내부 수정 X, public hook 호출만**.
- chat→form 흐름: draft(TaggedField) → 사람 수락 → `usePrimaryWriter`로 Config 슬라이스 저장(operation 태그 `save-character`/`save-world-sim`/`save-scene-direction` 기존 그대로) → shadow-logger가 자동 관측.
- 절대금지8: `studio-types`·`save-engine/*`·`origin-migration`·`useOriginTracker`·`OriginBadge`·`AuditExportButton`·`markdown/project-serializer`·`ManuscriptView` = **참조·hook 호출만, 0byte 수정.**

- **재framing 적용**: `_공통-3기둥.md` + `README.md`에 융합 원칙 반영(이번 점검에서 수정). 각 탭 doc은 착수 시 "기존 자산 → 융합" 절을 §4 표 기준으로 채움.
- **03(구성)은 이중 취약**: 소스 경로 broken(정밀성) + 기존 매핑 미확인(연계성) → firsthand 재검증 전 착수 금지.
- **순효과**: 융합 설계는 구현량을 크게 줄임 — relations·origin·coherence·save가 이미 있으니, 진짜 신규는 chat→form UI + 추가 메타 sidecar + QA 감사원 패널뿐.
