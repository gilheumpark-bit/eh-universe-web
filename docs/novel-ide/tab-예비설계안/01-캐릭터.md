## 핵심: 캐릭터는 3종 문서로 갈린다
세계관이 단일 entry였다면 캐릭터는 ① **CharacterEntry**(1명=시트 1장) ② **CharacterManifest**(작품 인구 인덱스) ③ **RelationGraph**(방향 비대칭 관계 엣지). 세계관과 가장 다른 점이 **커밋 단위가 3종**이라는 것.

## 타겟 스키마 (TS 스케치)

```typescript
interface CharacterEntry {
  // front-matter (세계관 WorldEntry와 평행)
  id: string;                  // "char_001"
  workId: string; type: "character";
  name: string; aliases?: string[];
  source?: string;             // 권위본 § (세계관과 동일 출처 추적)
  role: "protagonist"|"antagonist"|"supporting";
  tier: "T0"|"T1"|"T2"|"T3"|"T4"|"T5";   // ★ 양식 깊이 결정 (세계관 tier_depth 대응)
  firstAppearance: number; deathEpisode: number|null;  // ★ 회차 좌표 (세계관엔 없는 축)
  lockHistory: LockEvent[];    // ★ 화차별 확정 이력 (세계관 lock 대응)

  // 본문 — 세계관 Tier1/2/3에 대응
  surface:  { traits; appearance; publicPersona; imagePromptCore };       // ≈ Tier1 표면(외면)
  interior: { dna:{tier1:{desire,need,ghost,flaw}; tier2:{values,moral,psych}};
              mckeeContradiction; big5; attachment; pressureChoice };     // ≈ Tier2 내면·심리
  deep:     { abilities; motivationLayers; lifeStageArc; traumaEvolution };// ≈ Tier3 능력본질·생애

  // ★ 제4의 직교 축 — 세계관엔 없음. VOICE_PROTECTED (자동변환·자동grep·감사지적 금지)
  voiceDNA: { voiceFingerprint; obsessionKeywords; signaturePhrases;
              verbalTics; defaultEnding; speechLevel; voiceEvolution[] };

  relationships: RelationEdge[];   // 상세는 RelationGraph로 분리
  selfCheck: { spec; impl; core; total; grade };
}

interface RelationEdge {           // ★ 방향 비대칭 (A→B ≠ B→A) — 영구 lock
  fromCharId: string; toCharId: string;
  trustScore: number;              // -100~+100
  archetype: "love"|"hate"|"revenge"|"trust"|"betrayal"|"redemption";
  greimasRole: "Subject"|"Object"|"Sender"|"Receiver"|"Helper"|"Opponent";
  currentEpithet: string;          // "강민우 → 너"
  trustHistory: { episode; trustScore; triggerEventId; delta }[];
}

interface CharacterManifest {      // 작품 인구 인덱스 (파생 가능)
  workId: string;
  populationStatus: Record<Tier, number>;     // T0~T5 인구 통계
  episodeMatrix: Record<string, Record<Tier, string[]>>;  // 회차×캐릭 (시간축 인덱스)
  contrastMatrix: ContrastRow[];   // 5차원 시각 분리 (powerPosition/posture/gaze/space/symbol)
}
```

## Tier = 양식 깊이 (과잉/미달 차단)
T0=DNA풀+voice 17슬롯+관계매트릭스 / T1=DNA tier1+voice 기본 / T2=시그니처만 / T3=직업·위치만 / T4=집단 / T5=desire+flaw+소멸패턴. **T3에 풀 DNA = 과잉(YAGNI), T0에 시그니처만 = 미달.**

## 커밋 단위 (3종)
1. **캐릭터 1명** (가장 흔함) — Tier에 따라 깊이 가변.
2. **관계 1방향** — A→B 한 방향이 1 커밋. B→A는 별도. (tunnel-yuju에 강민우↔유주 관계 단독 커밋 실례.)
3. **인구 보강 1묶음** — T2/T3/T4 감초·기능·엑스트라.

## 채팅→폼 (실증)
tunnel-yuju `_작업_노트/캐릭_인터뷰_결정사항_2026-05-27.md`가 이 모델의 실물. 폼 필드마다 **provenance 라벨 강제**: `author_set`(작가 직접) / `ai_suggested`(AI 추천+근거) / `confirmed`(작가 확정) / `pending`(미확정). AI 채운 칸은 confirmed 전까지 "추천" 시각 구분. 확정 시 시트에 `★ 작가 확정 {날짜}` 인라인 lock.

## 정합검사 (캐릭터 고유 — 공통 감사원 A에 더함)
1. 이름 오타·호칭 일관성 (본문 vs `aliases`·호칭 진화 화차 lock)
2. 특성 모순 (traits vs 본문 행동, Big5 vs 압박선택)
3. **관계 방향 비대칭 위반** = 결정론 하드룰. `A→B = B→A` 자동 동일화 = 금지(−15, ★★★).
4. Trust Score 점프가 trigger_event 없이 발생 = 위반.
5. **차별화 거리** (Layer 106): traits 자카드<0.6 · voice cosine<0.5 · 외형<0.7 · flaw<0.6. 초과 경고 — 단 `intentional_overlap` 명시 시 면제(도플갱어 라이벌 등).
6. 인구 검증: 회차 매트릭스 T2/T3/T4 등장 0 → "생활감 부족" 경고(씬시트 연동).
7. voice drift: ep1 대비 >15% 알림, >50% 작가 검증.

## 인덱스 (3중)
세계관이 단순 refs 그래프라면 캐릭터는: (a) **방향 비대칭 가중 directed graph**(노드=캐릭터, 엣지=RelationEdge, 가중치=trustScore, 라벨=6관계유형) — ASCII Sociogram / 16×16 매트릭스, (b) **회차×캐릭터 매트릭스**(시간축), (c) **차별화 거리공간**(자카드/cosine).
