## 핵심: 집필은 세계관식 "폼 채우기"가 아니다
다른 탭(세계관·캐릭·씬시트·구성)은 **필드 채우기 폼**. 집필은 **"얇은 메타 래퍼 + 100% 자유 본문 + 사후 grep 측정"**. 본문 안에 필드를 박지 않음 — 메타(자수·대화비율·후크·beat·AI티)는 본문에서 **추출해 별도 로그**로. + **3스테이지 순차 파이프라인**(집필→퇴고→출고)이며 권한이 직교. 커밋 단위 = **1화**(.5화 인터루드 포함).

## A. 집필(원고) 탭 — "메타 래퍼 + 자유 본문" 2층

```
┌ [폼: 헤더] ───────────  # {N}화 — {제목}  (씬시트 title 자동 추출)
├ [자유 본문] ──────────  100% 산문 (대사 "…", 지문, 장면구분 ***). 필드 0개.
│                         ≈12.9K 문자 / clean ≈6.4K 음절 (M tier)
└ [폼: 인계 블록] ───────  *** [인계 사항] 회차·3축 곡선 정합·voice 준수·체크 PASS
```
- **폼 = 본문 바깥** (헤더 1줄 + 말미 인계 receipt). front-matter YAML 없음.
- **메타는 사후 측정치** → 본문에서 grep 추출, 별도 로그 파일(`_serial_memory_log`·`_payment_hook_log`·`_humor_log`·`_platform_revision_log`).
- **5-Part = 본문 서사 골격**(권장, 강제 X): ①기(13-17%)·②승(22-25%)·③전(22-25%)·④결(25-30% 최장)·⑤훅(10-15% 100-200자). 결제후크 stage_1~5 매핑.
- 생성 3모드(택1): B 통생성→파트진단(기본) / A 파트분할→통합 / Triple-Draft 3안 합성 / 4캔버스 누적(AUTO). ⚠ **파트별 통과 ≠ 전체 통과 → 조립 후 전체 게이트 재검증 강제.**
- 파일 2변형: `epNNN_final.md`(본문+인계블록) / `epNNN_final_clean.md`(순수 본문). **퇴고·출고는 clean만 받음.**

```typescript
interface Manuscript {
  episodeId: string; title: string;     // 폼 = 헤더만
  body: string;                         // 자유 산문 (필드 없음)
  handoff?: { curveCheck; voiceCheck; checklistPass };  // [인계 사항] receipt
  // 메타는 본문에 없음 — 별도 측정 로그로 추출:
}
interface ManuscriptMetrics {           // grep 추출 (본문 외부 로그)
  charCount; dialogueRatio; endingDist; aiSignatureScore;  // 0~100
  hookType; partRatios: number[]; paymentHookStages;
}
```

## B. 퇴고 탭 — 6-STEP 진단 (본문 수정은 STEP6만)
근거 `_웹소설_정제_6단계.md`. 사상: **"한 번에 한 가지만"** (통째로 "고쳐줘" 금지 — AI가 자기 스타일로 덮음).
`⓪ 반복분석 → ① 쓰레기제거 → ② 교정 → ③ 교열 → ④ 구성편집 → ⑤ 윤문 → ⑥ 최종검수`
- **STEP 1~5 = 진단 보고서만**(본문 수정 시 −15점). **실제 수정 = STEP 6**(작가 "퇴고본 수정 ep{N}" 명령 시).
- 본질 = **자수 맞춤**: 모자라면 채움(묘사·심리·정지·이동) / 과하면 삭제(잉여·반복·AI시그니처·인포덤프). **삭제=채움 1:1**(수정 후 자수<전이면 차단). 채우기 금지(새 설정/캐릭), 지우기 금지(핵심사건·voice·Magic Moment·사이다 3박자).

근거 `_출고_단계_표준.md`. 4영역: 가시성(단락 3-4줄·줄바꿈 30-50·벽돌체 차단)·호흡(퇴고 lock)·조판(장면전환 `---`·대사 시각배치·마크다운 100% 제거)·텐션(퇴고 lock). **출고는 가시성·조판만, 호흡·텐션은 퇴고에서 완료. 본문 voice ±0%.**

## 정합검사 (생성 후 자동 grep — 4종 직교)
1. **자수** — 플랫폼 임계 ±5%(카카오 5,500-6,500/문피아 5,000+/노벨피아 4,500-5,500…), M tier 5,500-7,000 미달 −20점, 삭제=채움 1:1.
2. **AI티** — 13항목 시그니처 점수 0-100(대시 11+·시간정밀 5+·음성톤 2+·마크다운/이모지 즉시 −5). 인간 baseline: `~다`어미 80-92%(92%+ AI의심)·대화 10-45%·단문 25-40%.
3. **금지 패턴** — 63매트릭스 + **단정형 정리문 5종 1화1회**(2회 −5/3회+ −10) + 고빈도 4단어(지독하게·기묘한·논리적·매끄러운) 회차당 1-2.
4. **플랫폼 가독성** — 7플랫폼×7영역 49매트릭스.
→ **강제 차단 X**(voice DNA 우선). 순수 기계결함(마크다운·이모지·줄깨짐)만 강제. PASS/CONSIDER/REVIEW/CHECK 4등급. (= 공통 감사원 A에 통합.)

## 인덱스 (화 시퀀스)
`_serial_memory_log`: 기억 decay 3단계(Hot 1-2일 90%/Cool 1주 60%/Cold 1달 30%), 복선 setup-payoff 거리(3-7화 성공/30화+ Hot Recap 의무), 결제 마디 7 KPI(1·2·3-5·10·25·50·100화), 5화 클리프 variety. → 화 시퀀스 = 복선거리 + 결제마디 + 클리프 다양성 인덱스.

## 채팅→폼
세계관식 "필드 채움" 아님. 채팅 브레인스토밍 → **5파트 본문 생성**(B 통생성 기본) → AI가 인계 receipt 자동 첨부. 폼 채움 대상 = 헤더 제목(씬시트 추출) + 사후 측정 로그뿐. **본문은 AI 생성 + 사람 검토/커밋.**
