# PART 0 — 공통 기반: WorldEntry 스키마 (= 안③의 코어)

tunnel-yuju 양식을 역설계한 타입. **격리**: `studio-types.ts`(절대 금지 8파일)는 0byte. 신규 모듈 `src/lib/worldgraph/` 자체 타입.

```ts
// src/lib/worldgraph/types.ts  (신규 — studio-types 미수정)

export type EntryType =
  | 'world_fact'              // 일반 세계관 사실 (대부분)
  | 'integration_matrix'      // 정합 매트릭스 (안② 룰 소스, priority 0)
  | 'checklist_audit'         // 체크리스트 감사 결과
  | 'tone_guide';             // 집필 톤 가이드

export type SpoilerClass = 'public' | 'restricted' | 'confidential';
export type EntryStatus  = 'draft' | 'canon' | 'archived';

export interface EpisodeRange { from: number; to: number | null; } // to=null → 이후 전 영역

/** 잠금 이벤트 = "확장마다 커밋". tunnel-yuju 의 ★★★★★ + 날짜 + 작가 확정. */
export interface LockEvent {
  date: string;        // ISO (args 로 주입 — 스크립트 Date.now 금지 규약과 무관, 런타임 UI는 OK)
  stars: 1|2|3|4|5;    // ★ 강도
  note: string;        // "작가 확정 …"
  author: string;
}

/** tier 본문 블록 — 산문 / 잠금 yaml / 참조 포인터. */
export type Block =
  | { kind: 'prose'; text: string }
  | { kind: 'lock';  stars: 1|2|3|4|5; date: string; title: string; body: string } // ```yaml 블록 원문 보존
  | { kind: 'ref';   targetId: string; section?: string; label: string };           // → 풀 영역: x.md §N

export interface EntryRef { targetId: string; section?: string; kind: 'full' | 'related'; }

export interface WorldEntry {
  // identity
  id: string;                 // ULID (파일명과 분리 — rename 안전)
  workId: string;             // "tunnel-yuju"
  type: EntryType;
  domain: string;             // "tunnel / metaphysics"
  title: string;

  // provenance / canon
  source?: string;            // "권위본 §3" — SSOT 추적
  priority: 0 | 1 | 2 | 3;    // 0 = canon-critical / matrix
  locked: boolean;            // 영구 lock (사람만 해제)
  lockHistory: LockEvent[];   // 커밋 로그

  // spoiler / exposure
  tierDepth: 1 | 2 | 3;
  spoilerClass: SpoilerClass;
  exposure: EpisodeRange[];   // 노출 영역 (화)

  // content (tiered)
  tier1?: Block[];            // 표면
  tier2?: Block[];            // 시스템
  tier3?: Block[];            // 형이상학 (스포일러)

  // graph
  refs: EntryRef[];           // 교차참조

  // meta
  status: EntryStatus;        // 사람 커밋 = canon
  createdAt: string; updatedAt: string;
}
```

설계 결정:
- **파일명 ≠ id**: tunnel-yuju 는 파일명이 곧 키였다(`_world_터널_구조.md`). rename·이동 시 깨짐. id(ULID) 분리 + 파일명은 `slug` 메타로.
- **`lock` 블록은 yaml 원문 그대로 보존**: tunnel-yuju Tier3 의 ```yaml``` 잠금 블록을 과파싱하면 손실. 원문 문자열로 들고 round-trip.
- **matrix·checklist 도 WorldEntry**: 정합 매트릭스(안②의 룰)·체크리스트가 별도 타입이 아니라 `type` 만 다른 entry → 같은 그래프·같은 커밋·같은 인덱스에 올라탐.

# PART 1 — 안③: 양식 타겟 + 직렬화 (round-trip)

**목표**: tunnel-yuju `.md` ↔ `WorldEntry` 무손실 왕복. = 사용자가 말한 "읽어오기·내보내기 그 양식에 맞게".

```
src/lib/worldgraph/serializer.ts
  parseEntry(md: string): WorldEntry        // YAML front-matter + 본문 → 구조화
  serializeEntry(e: WorldEntry): string     // → tunnel-yuju 양식 .md (front-matter + Tier + 노출표 + 관련fact)
```

파싱 규약 (tunnel-yuju 실측 기준):
- front-matter `---` 블록 → identity/provenance/spoiler 필드. `tier_depth: 3 (...)` 같은 주석 꼬리 허용 (정규식 선행 숫자만).
- `## Tier N …` 헤더 → tier1/2/3 분기.
- `### ★★★★★ … NEW <date>` + ```yaml``` → `lock` Block (stars=★수, date 추출, body=원문).
- `→ 풀 영역: x.md §N` / `→ 풀 영역: x.md` → `ref` Block + `refs[]`.
- `## 노출 순서` 표 → `exposure[]` (구간 행 파싱).
- `## 관련 fact` 리스트 → `refs[] kind:'related'`.

**검증 게이트 (안③ Done 정의)**: 기존 tunnel-yuju 31 파일 전부 `parse → serialize → 원본 diff`. **의미 무손실**(공백·정렬 차 허용, 내용/순서/잠금/참조 0 손실). 이게 통과해야 스키마가 맞다는 증명 — Phase 0 의 단일 합격선.

저장: `worldgraph/store.ts` — IndexedDB. **N-01 교훈 적용**: DB 스키마는 `shadow-db-schema.ts` 처럼 **단일 SSOT 모듈**(`worldgraph-db-schema.ts`)에서 버전+store 일원화. 4모듈 drift 재발 금지. + GitHub 동기화는 기존 `github-sync.ts` / `project-serializer.ts` 경유로 `.md` 라운드트립.

# PART 2 — 안②: 채우기 / 정합 분리 (적대적)

**핵심 제약 (tunnel-yuju 매트릭스가 직접 박아둔 것)**:
> "LLM 본능 = 영역 합치기 본능 영구 한계" / "민우 = 수오 → -20점"
> **폼을 채우는 AI가 곧 합치려는 본능의 AI.** ⇒ 채우기와 정합검사는 반드시 **분리된 적대적 패스**.

## 2.1 두 패스

```
[채팅 brainstorm]
   │
   ├─ FILL (Composer 역할, temp 보통) ─────────────────────────────
   │     입력: 채팅 transcript + 대상 entry(신규 or 기존) + 도메인
   │     출력: WorldEntry draft (tier 채움 + refs 제안)
   │     구현: generateJsonViaSpark (stream:false → WorldEntry JSON)
   │
   ├─ COHERENCE (Auditor 역할, temp 낮음, **별도 호출**, 적대적) ──
   │     입력: draft + 기존 canon 그래프 요약 + coherence 룰셋(매트릭스 entry)
   │     프롬프트: "동의하지 말고 위반을 찾아라" (refute 지향)
   │     출력: Violation[]
   │     구현: 별도 generateJsonViaSpark (tier-registry Auditor 개념 / CLAUDE.md self-critique 세션)
   │
   └─ 사람: draft + 필드별 Violation 플래그 → 수정/override/떡밥표시 → COMMIT(lock+canon)
```

왜 분리: 같은 호출이 채우고 검사하면 자기 위반을 못 본다(tunnel-yuju "자기 위반 정직 인정"이 그 증거). Auditor 는 매트릭스 -20 룰을 **하드 데이터로** 컨텍스트에 받고 refute 지향 + 저온. priority 0 entry 는 **N-표 다수결**(적대적 검증, bug-hunter 패턴).

## 2.2 Coherence 룰셋 (3종 검사)

```ts
src/lib/worldgraph/coherence.ts
  checkCoherence(draft, canonGraph, matrix): Violation[]
```

| # | 검사 | 방식 | 출력 |
|---|---|---|---|
| C1 | **lock 위반** | `locked:true` fact 를 덮어쓰려는가 | **하드 블록** (사람 unlock 전 불가) |
| C2 | **forbidden-merge** | 매트릭스 금지쌍("민우=수오") 패턴/임베딩 매칭 | penalty(-20/-15) + 플래그 |
| C3 | **모순(contradiction)** | 기존 잠금 fact 와 의미 충돌 (예: "마법 무료" vs 잠금 "마법 과세") | LLM 판정 + 사람 확인 |

- C1·C2 = **결정론적**(룰/패턴) → 신뢰 높음.
- C3 = **LLM 판정** → best-effort, false pos/neg 존재 `[확인 필요]`. 과신 금지, 사람 검토 전제.

```ts
interface Violation {
  ruleId: 'C1-lock' | 'C2-merge' | 'C3-contradiction';
  severity: 'block' | 'high' | 'warn';
  field: string;            // 어느 tier/블록
  message: string;          // "draft tier3 가 '민우=수오' 합치기 시도 (-20)"
  evidenceEntryId?: string; // 충돌 상대 entry
  suggestion?: string;
}
```

## 2.3 떡밥 ≠ 결함 (사람이 종료)

체크리스트가 이미 구분: **"미결정 = 작가 영역/떡밥이지 결함 아님"**. ⇒ Violation 을 사람이 `intentional` 로 마킹하면 **예외로 기록** → 재플래그 안 함. AI 는 *표시*만, **떡밥/구멍 판정은 작가**. (= "종료는 사람"의 세계관판.)

# PART 3 — 안①: 인덱스 자가유지

**문제 (실측)**: tunnel-yuju README 는 "파일 인벤토리 **11건**"이라 쓰고 표엔 15개, 실제 폴더엔 **31개**. = 오늘 고친 `findings.json` 원장 드리프트와 동일 병. 손으로 인덱스를 못 따라잡음.

**원칙**: 인덱스는 **저장하지 않고 entry 컬렉션에서 파생**. 손편집 0 → 드리프트 구조적으로 불가. (M-01 교훈: 장부는 손유지 X, 파생 or 닫힘루프.)

```ts
src/lib/worldgraph/index-builder.ts
  buildIndex(entries: WorldEntry[]): WorldIndex   // 전부 derived
```

파생 뷰 (전부 자동):
1. **인벤토리 표**: title·domain·tierDepth·exposure·locked·refs수·source(권위본 §). README 표를 생성(손편집 금지).
2. **그래프 무결성**:
   - **깨진 ref**: `EntryRef.targetId` 미해결 → 플래그 (Novel IDE "Find References" 재사용).
   - **고아 entry**: in/out ref 0.
   - **백링크**: A→B 면 B 측에 A← 자동 노출.
3. **노출 타임라인**: 전 entry `exposure`+`tierDepth` 집계 → "화 N 시점, 어떤 entry 의 어떤 tier 가 public" 맵. = 스포일러 게이트 뷰(독자/편집자/작가 view 분기, creative-process view 개념 재사용).
4. **정합 대시보드**: 그래프 전역 open Violation 집계 (findings.json 의 세계관판, 단 derived).
5. **변경 피드**: 모든 `LockEvent` → changelog 자동(chg_W1 패턴 자동화).

저장 vs 파생: **SSOT = entries**. 인덱스는 read 시 계산(쓰기 시 invalidate). 절대 안 어긋남.
