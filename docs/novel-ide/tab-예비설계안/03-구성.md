# 03 · 구성 탭 (플롯·메인스토리·시놉시스) — 예비설계안 (대기)

## 핵심: 3개를 쪼개면 데이터 중복 → "구성" 단일 탭
시놉시스는 별도 데이터가 아니라 **메인스토리의 파생 출력**(내부 `_premise` → 외부 `_synopsis` 추출). 플롯 구조는 메인스토리의 하위 슬롯. → **"구성(Composition)" 단일 탭 = [메인스토리 12영역] + [플롯 구조 섹션] + [시놉시스 출력 뷰]**. 장르는 작품 메타 필드 + 읽기전용 baseline 패널.

## 타겟 스키마 (TS 스케치)

```typescript
interface MainStory {                  // 작품당 1개
  meta: { workId; type:"main_story"; version; locked; source?; spoilerClass?; date };
  soul: string;                        // §1 작품 영혼 (theme 상위 한 문장)
  synopsis: Synopsis;                  // §2 — 시놉 탭의 원천 (아래)
  plotStructure: PlotStructure;        // §3 (아래)
  seasonBranch?: { canon; extension?; ifLines? };  // §4 평행우주 분기
  mainOutline: string;                 // §5 큰 줄기
  chapterIndex: ArcNode[];             // §6 아크 인덱스 (3~12 아크)
  mainEventChain: { ep; event; type; impact }[];   // §7 핵심사건 인과 체인
  characterArcs: { charId; tier; selfRevelationEp?; arc }[];  // §8 Tier별 변화곡선
  foreshadowGrid: ForeshadowEntry[];   // setup→payoff 매트릭스
  toneProfile: { base; actionRatio; dialogueRatio; innerRatio; ... };  // 실물 60/30/10
  endingLock: { style:"hybrid"|"full_closure"|"bitter"|"open"; locked: boolean };
}

interface PlotStructure {
  primary: PlotPatternId;              // 25패턴: hero-journey / save-the-cat / korean-regression ...
  episodeCount: number;                // 25 / 100 / 400
  beatMapping: Record<string, { beat; role; desc? }>;  // "ep001" | "ep008-012" → 비트
}

interface ArcNode {                    // 인덱스 단위 (_arc_structure.md 통합, 개별 _arc_NN 생성 금지)
  arcId: string; title; episodes: string;  // "1-25"
  theme; keyEvent; cliffhanger?;
  fourPhase?: { setup; rising; climax; resolution };
}

interface Synopsis {                   // 외부 제출 7영역 (메인 §2에서 파생)
  title; genre:{primary;sub?;keywords[]}; intent;  // 기획의도 = 차별점 3~5건 강제
  logline: string;                     // 갈등+목표+장르시그널 / 결말누출·메타언어 금지
  characters: { protagonist; supporting[]; antagonist };
  worldview: string;
  plotSummary: { gi; seung; jeon; gyeol; endingExplicit: boolean };  // 제출용 결말 명시 강제
}

interface ForeshadowEntry { id; setupEp; payoffEp; distance; recapRequired? }
```

## 커밋 단위 (3계층)
1. **메인스토리 = 작품 1회 커밋** (시작 시 12영역 한 번에, 이후 마디 10·25·100화마다 정합 재검증).
2. **아크 = 중간 커밋** (arc_01 = 1~25화 묶음).
3. **화 씬시트 = 최소 반복 단위** (실제 "채팅→폼→커밋" 사이클은 1화 단위 — 02 탭).

→ 구성 탭은 **상위 골격(작품·아크)을 굵게 박고 마디마다 갱신**, 회차 반복은 씬시트 탭이 담당.

## 장르 baseline (읽기전용 패널 + 검사 기준)
KPIPA 2024: 시장 1.35조 / 네이버+카카오 84.5% / 인당 35.7화. 작가 4 tier(<500만 70.8% … 1억+ 1%) → 모드 권장(AUTO→FULL). **초반 5화 강제 리듬**: 1화=질문(hook 200자)·2화=행동(tension 35-50)·3화=보상(도파민1+)·4-5화=규칙(텔링 60% 초과 차단). genre×플롯 자동 매핑(헌터→Hero's Journey·회귀→7-Point·로판→Romance Beat). **단 baseline일 뿐 — BIBLE 맹신 차단, GUIDE NOTE.**

## 채팅→폼
모드 분기: **AUTO**(인터뷰 0·default plot=Save the Cat) / **GUIDED**(3-5질문: plot·아크수·톤·결말·복선핵심) / **FULL**(영역별 결정·복선 12+). 작가 명령 = 폼 트리거: `메인 스토리 구성` / `plot 구조 보고` / `아크 구조 잡기` / `시놉시스 출력` / `로그라인 진단`.

## 정합검사 (구성 고유)
- **플롯↔씬시트**: `ep{N}` 비트 = beatMapping[ep{N}] 정합? (전환점/적/캐릭 decision)
- **복선**: setup_ep→payoff_ep 거리, payoff_rate 0.6-0.9, 거리>10화 recap 강제.
- **결말**: endingLock ↔ arc_final ↔ 캐릭 self-revelation 일치.
- **장르 baseline 위반**: 초반5화 게이트·자수 표준·시놉 7영역 정량(로그라인 grep·차별점 등급·줄거리 결말명시).
- 자동 트리거: 10·25·50·100화 마디.

## 인덱스 (2계층)
**ArcNode**(arc_01~NN) → **EpisodeNode**(ep001~400 + ep{N}.5 인터루드). chapterIndex(§6)=아크 인덱스, mainEventChain(§7)=인과 그래프, foreshadowGrid=setup→payoff 엣지. 세계관 refs처럼 노드 간 참조 그래프(사건→아크→캐릭arc→복선 크로스링크).
