# 기둥 ② — 채우기 / 정합 분리 (적대적·비수렴) ★ 최중요

작가 시스템의 **QA 감사원**(`_QA_감사원_시스템.md`)이 이 기둥의 정확한 근거다. "폼 채우는 AI = 합치려는 본능의 AI"라 채움(generation)과 검사(audit)를 분리하되, 단일 Auditor가 아니라 **직무가 다른 비수렴 패널**로 한다.

## 감사원 패널 (업계 5원리 = 계층·지식분리·적대성·독립성·편향차단)

| 감사원 | 인식 상태 | 질문 | 핀셋 대상 | 발동 |
|---|---|---|---|---|
| **A 정합검사관** (white box) | 의도·설정 **다 앎** | 맞나 틀리나 | 설정모순·약속미회수·continuity·정량·표면 | 매 커밋 |
| **B 외부독자** (black box) | 설정 **모름**(본문만) | 재밌나 납득되나 | 납득·재미·이탈·이해 | 매 커밋 |
| **C 반증가** (red team) | 공격자 | 왜 망하나 | 구멍·클리셰·안믿김 | 매 커밋 |
| **D 구조검사관** (선택) | 거시 | 구조 서나 | 구조·주제·아크·페이스 | 아크/완결 마디만 |

## 운영 규칙 (전 탭 동일)
- **독립·비수렴**: A·B·C가 **각자 따로** list를 내고 서로 안 본다. "토론해서 하나의 결론"은 금지(그건 합의=리허설). B가 A의 설정을 참조하면 black box가 깨짐.
- **겹침/충돌 해석**: 2+ 감사원 겹침 = **확실한 문제**(우선). A PASS·B FAIL 충돌 = **"설명 부족" 신호**. 단독 = 작가 판정.
- **점수 합산 X, list 대조만** (편향이 점수에 섞임).
- **발견까지만 — 고침은 작가**(영구 lock). 모든 지적 = 권고. 작가 "voice다/의도다/무시" 시 폐기.

## 프로젝트 하드룰 = A의 결정론 입력
세계관 정합 매트릭스의 **"민우=수오 → −20점 ★★★★★ 합치기 금지"** 같은 작가 정의 금지쌍은 A 정합검사관의 **결정론 룰 입력**(패턴 매칭). LLM 판정(B·C)과 분리. 즉 C2(forbidden-merge)는 결정론, C3(의미 모순/납득)는 LLM.

## 정직 한계 (반드시 명시 — anti-sycophancy)
- **같은 LLM = 공통 사각 공유**: A·B·C가 같은 모델이면 "모델이 원래 못 잡는 것"은 다관점으로도 못 잡음. B(black box)도 같은 가중치 시뮬 = **모방된 외부성**(진짜 외부 X).
- 핀셋 정확도: 표면·정량(A) 0.8–0.9 / 깊은 의도(A) 0.55–0.7 / 독자 체감(B) 0.5–0.6 / 구멍(C) 0.55–0.7.
- **진짜 독립은 다른 모델·실제 사람.** 고-위험 커밋(priority 0)은 다른 모델 또는 사람 1패스 권장. 본 시스템은 근사.

## eh-universe-web 매핑
- 채움(Composer) = `generateJsonViaSpark` (stream:false → 폼 JSON)
- 감사원 A/B/C = **각각 별도** `generateJsonViaSpark` 저온 호출 (B는 설정 컨텍스트 차단, C는 refute 프롬프트). `tier-registry` Auditor 개념 / CLAUDE.md self-critique 세션.
- 결정론 A 입력 = 프로젝트 매트릭스 entry + `useContinuityCheck` / worldbook-violation.

# 기둥 ① — 인덱스 자가유지 (파생·원본/파생 이원화)

근거: `_권위본_자동_갱신_trigger.md` + `_종합_파일_표준.md` + `_enforcement_manifest.md`.

## 원본 ↔ 파생 이원화
- **원본** = 구조화 entry (YAML front-matter + 슬롯). **AI 1순위**.
- **파생** = 자연어 종합 인덱스(작가가 책처럼 읽는 뷰: 세계관 바이블/인물 가이드/회차 outline/AI 제작 노트 등 7종). **작가 1순위**.
- 산출물 Write 직후 원본 → 파생 **자동 변환·append**. 작가가 파생을 직접 고치면 → AI가 원본 메타 **역갱신**.

## 자가유지 메커니즘 (드리프트 구조적 차단)
- **인덱스는 저장 안 하고 entry에서 파생**(README 인벤토리·노출 타임라인·정합 대시보드·changelog). 손편집 0 → tunnel-yuju README "11건 vs 실제 31건" 같은 드리프트 불가.
- **권위본 자동 갱신 trigger**: 변경 박힘 직후 권위 인덱스 cross-ref grep → 불일치 자동 patch. (정직: 70–80% 자동, 100% X.)
- **enforcement_manifest = 중앙 오케스트레이터**: 매 산출 직후 존재 확인 → 검증 → 자동 로그 갱신 → 종합 동기 → 1줄 요약. 분산 금지, 단일 참조점.
- **N-01 교훈**: IDB는 단일 스키마 SSOT 모듈(`*-db-schema.ts`)로. 모듈별 스키마 복제 drift 금지.

## eh-universe-web 매핑
**신규** `*/index-builder.ts`(derived) + `*/serializer.ts`(.md 라운드트립) + `github-sync.ts`(동기화·기존) + IndexedDB **단일 스키마 SSOT 모듈**(N-01 교훈).
⚠️ `project-serializer.ts`·`markdown-serializer.ts`는 **절대 금지 8파일** — **참조만, 수정·재사용 X**. 라운드트립은 신규 serializer가 담당.

# 공통 — 커밋·영수증·게이트·voice

## 작업 영수증 (커밋마다 자동 발급)
근거: `_작업_영수증_표준.md`. 8단계 산출물 Write 직후 자동: **✓ 한 것(구체 증거: 파일·줄번호·grep 수치·인용)** / **✗ 안 한 것(사유)**. 추상 "✓ 적용함" 금지. "누락 없음/100%" 단정 = 감점, ✗ 섹션 생략 = 영수증 무효.
→ **검증=read-only(본문 무수정 자동) / 본문 수정=작가 명령**. 이 권한 경계가 ②(채움 vs 검사)를 가르는 실제 선.
→ eh-universe-web의 **창작 과정 확인서(증인)** 작업과 직결 — 영수증이 곧 과정 기록.

## Cycle 진입 게이트 (시작 게이트)
근거: `_Cycle_진입_게이트.md`. 이전 단계 PASS 없이 다음 진입 X. 차단 시 작가 1회 결정("보강/강행/건너뛰기") — 강행 시 로그에 작가 책임 기록. AI 자기 합리화("이번엔 건너뛰기") 차단.
→ **종료=사람 커밋** 원칙의 시작측 대칭: 진입도 게이트, 통과 결정권은 작가.

## BIBLE 맹신 차단 (= NOA anti-sycophancy 정합)
근거: `_BIBLE_맹신_차단_protocol.md`. **BIBLE(절대권위) → GUIDE NOTE(참고)**. AI 맹신 패턴(권위 명명·자동 도식화·단정) 감점. 시스템 룰조차 "참고", **작가 결정이 영구 상위**.

## voice DNA = VOICE_PROTECTED
근거: `_voice_DNA_자동적용.md`. 작가 voice(어휘·리듬·signature·obsessionKeywords)는 **자동 변환·자동 grep·감사원 지적 침범 금지**. voice는 "어긋남"이 아님.

## 작가 부담 모드 (Progressive Disclosure)
근거: `_작가_부담_모드.md`. **AUTO**(인터뷰 0·천장 60–75) / **GUIDED**(5–7질문·78–88) / **FULL**(20–30질문·92–98). "어느 모드?" 1회 묻고 따름. enforcement는 AUTO에서도 백그라운드 자동(부담 X), 결과만 1줄 보고.
→ eh-universe-web `UserRoleContext`(explorer→AUTO / writer→GUIDED·FULL)로 분기. 파워유저 폼 직행 = AUTO 우회.
