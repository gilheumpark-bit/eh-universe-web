# Loreguard Design Principles

> 작가가 하루 6시간, 6개월, 6년을 머무는 집.
> 장편 한 권은 평균 800–1200시간의 집중 작업. 이 시간 안에 몸이 상하지 않고,
> 세계관이 흐트러지지 않고, 작가가 주어로 남는 단 하나의 작업실.

## 세 기둥

1. **장기 체류 (Long Dwell)** — 세션 6–8시간, 프로젝트 6–24개월. 화면은 정적, 상태는 유지, 상시 노출 정보는 최소.
2. **인체공학 (Ergonomics)** — 눈·목·허리·마음을 모두 설계 대상으로. 대비·색·움직임·밀도·모드를 작가의 하루 시간대에 맞춰 조정 가능.
3. **작가가 주어 (Writer as Subject)** — 시스템의 자랑(수치·속도·규모) 숨김. AI는 "답"이 아니라 "자료". 모든 자동은 끌 수 있다.

## 원칙 R1–R8

| # | 원칙 | 요지 |
|---|---|---|
| R1 | Resting Rectangle | 본문 중앙 영역은 절대 움직이지 않는다. 본문 가로 최대 68ch (`--editor-max-width`). |
| R2 | 3-Depth Rule | 작가가 자주 쓰는 기능은 3-클릭 이내. 4번째 클릭부터는 희귀 기능. |
| R3 | Header Recession | 본문에 커서 들어가면 헤더·상태바 `opacity: 0.3` 로 물러남. |
| R4 | Font Weight ≤600 | Display 서체는 600 최대, 본문 400–500. Bold(700)는 UI 거의 금지. |
| R5 | Soft Caret | Caret 색상은 amber 톤(검정/흰 회피). blink 주기는 브라우저 한계로 커스터마이즈 불가. |
| R6 | Single Popover | 어떤 순간에도 오버레이는 최대 1개. 추가는 queue. |
| R7 | Inline Everything | 설정·검색·제안·코덱스 조회는 가능한 사이드 패널/인라인. 모달은 파괴 동작 전용. |
| R8 | Chosen Density | Density는 사용자 선택(cozy/compact/spacious). 한 번 설정 후 영속. |

추가: **R11 IME Silence** — compositionstart~end 동안 자동저장·자동제안·자동분석 훅은 침묵. 한글/일본어/중국어 조합 중 Tab은 조합 확정이어야 한다. (구현: `src/lib/ime-guard.ts`)

## 부록 — 장기 운영 고려사항 (별도 결정 대상)

- 문서 수명 (프로젝트는 앱 버전보다 길다 → 내보내기 1등 시민, 인간 가독 포맷)
- 데이터 주권 (어디에 저장되는지 작가가 눈으로 봄, 오프라인 무결성)
- 복원 (Undo 크로스 세션, 삭제 "휴지통 아닌 서랍" 30일)
- 이름 안정성 (지명·인물 철자 전역 리팩터, 깨진 참조 감지)
- 세션 건강 (2시간 연속 → 조용한 휴식 제안, 22시 이후 자동 dim)
- 소리 디자인 (기본 무음, ambience 플레이어 옵션)
- 인쇄 표면 (원고지 / Times 12pt 더블 스페이스 프리셋)
- 탭 간 상태 연속성 (마지막 본 위치 영속, 선택 동기화, 뒤로 가기)
- 접근성 장기 특이 (노안 프리셋, 한 손 모드, 스크린리더 본문 읽기)
- 프로젝트 복수성 (시리즈물 코덱스 상속, 프로젝트 스위처 1초)
- 협업 경계 (한 작가 + 편집자·번역자·베타리더는 "손님")
- 비밀 모드 (AI 인덱싱 차단, 로컬 백업만)
