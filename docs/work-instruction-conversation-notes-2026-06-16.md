## 작업 노트

### 완료된 핵심 작업

1. 사용자 노출 문구 클리닝 1차 진행
   - 공개 UI 기준으로 `API 키`, `BYOK`, `AI 생성`, `AI 채팅`, `Gemini` 같은 직접 노출 표현을 줄이는 방향으로 정리.
   - 대체 기준: `연결 키`, `노아`, `노아 제안`, `과정기록`, `출고 패키지`, `권리/IP 점검`.

2. 프로젝트 시작 화면 UX 재정리
   - `새 작품`, `최근 작품`, `파일에서 가져오기`가 같은 화면처럼 보이던 문제를 구조적으로 구분.
   - 작품 보관함, 파일 불러오기, 기준 입력, 충돌 후보의 의미를 더 직관적으로 정리.

3. 출고 화면 의미 재정리
   - `출고`를 돈/결제 중심으로 먼저 보이게 하지 않고, 작가에게 남는 산출물과 제출 목적을 먼저 보이도록 정리.
   - 가격/크레딧은 하단 조건 정보로 낮춤.

4. Firebase 로그인 복구/확인
   - Firebase 공개 키 성격은 사용자와 합의: 공개 설정값은 클라이언트에 노출되어도 되는 값이다.
   - 실제 로그인은 브라우저에서 `127.0.0.1:3000` 도메인 등록 문제로 막혔고, Authorized domains 추가 후 사용자가 로그인 화면 확인.
   - 이후 Codex가 웹페이지 클릭하면 앱이 꺼지는 문제가 있어, 가시 브라우저 조작은 피하고 headless 검증 선호.

5. 권리/IP·저작권 등록 패키지 설계 반영
   - 사용자 실제 한국저작권위원회 보완 요청 사례를 설계 근거로 반영.
   - A안 서사 중심, B안 캐릭터 중심, C안 추상 주제 중심 등록 패키지 방향 확정.
   - 저작물 설명에는 주제, 줄거리, 인물, 주요사건, 갈등요소, 표현상 특징을 채워야 함.
   - 제출 복제물에는 보호범위/권리 귀속 주장 문구를 과하게 넣지 않는 방향.

6. 계약 전후 어드바이스/권리 자산화 니치 재정의
   - 계약 협상 코치는 대놓고 법률 대리처럼 쓰는 기능이 아니라, 작가가 계약 전후에 업계 표준 대비 문서를 분석하고 대응 전략을 세우는 기능.
   - 사용자의 사업 니치: 작가의 이득을 권리화하고, 작품 성공 후 계약·제출·권리/IP 판단에 쓸 근거를 축적하는 것.
   - 지나친 회피성 설계보다 작가가 실제 결제할 이유를 만드는 실용적 패키지를 우선.

## 대화 노트

### 사업 방향

사용자는 로어가드를 단순한 자동 집필 도구로 포지셔닝하지 않으려 한다.

핵심 포지션:

- `창작의 IDE`
- 한국 웹소설/연재 작가 현실에 맞춘 도구
- 작가가 직접 고민하고 만든 과정, 세계관, 권리/IP 판단을 자산화하는 도구
- 생산이 쉬워진 시대에 작가의 고유한 판단과 과정 자체를 프리미엄 자산으로 만드는 제품

사용자가 싫어한 방향:

- 자동 집필 툴처럼 보이는 표면
- 안전 타령만 하며 실질적 결제 명분을 만들지 못하는 기능
- 법적 책임 회피 문구가 제품의 자신감을 죽이는 설계
- 의도적 마찰을 UX 허접함의 변명으로 쓰는 것

사용자가 원하는 방향:

- 작가가 “이 기능이 있고 없고 차이가 있다”고 느끼는 것
- 초급/중급/고급 작가가 각자 결제 이유를 느끼는 구조
- Ultra 30만원 결제 명분이 되는 권리/IP 패키지
- 작품 등록 후 시간이 지나 성공했을 때 계약 전후 어드바이스가 이어지는 흐름
- 업계 표준, 실제 등록 사례, 후기, 보완 요청 사례를 반영한 실전형 패키지

### 창업/커뮤니티 관련

- 사용자는 `모두의 창업` 1차 합격 후 오리엔테이션 참석.
- 정부 바우처가 실제 해외 API/개발 인프라 비용에 쓰기 어렵다는 점에 실망.
- 커뮤니티 글 초안에서 AI 출력 티가 나는 `#`, `별표`, 과한 문어체 제거 요청.
- Discord 링크는 최종적으로 `https://discord.gg/vc3eZHkUh`
- 닉네임 규칙:
  - `[작가]별명`
  - `[독자]별명`
  - 기타 직군도 같은 형식 포함
  - 규칙 미준수 시 강퇴 가능

### 권리/IP 패키지 관련

사용자가 제공한 실제 저작권 등록 보완 요청의 핵심:

- 저작물 내용설명 보완 필요
- 권별 주제, 줄거리, 인물, 주요사건, 갈등요소 기재
- 연재목록 캡처는 제출 대상 아님
- 등록 원하는 저작물 그 자체를 업로드
- 세부 종류는 `어문저작물 > 기타`가 필요한 경우 있음
- 파일 내 작성일과 창작연월일 불일치 주의
- 외국어 제호는 국문 해석 또는 발음 병기
- 표현상 특징, 서술구조, 문체, 주제를 중심으로 설명
- 복제물에서 저작권 귀속 및 보호범위 주장은 삭제 요청 가능
- 필명/이명 확인 문구 필요 가능
- 보완 후 반드시 `신청` 버튼까지 눌러야 완료

설계 방향:

- A안: 서사 중심 등록 패키지
- B안: 캐릭터 중심 등록 패키지
- C안: 추상적 주제 중심 등록 패키지

### 확인서/추적 기술 관련

확인서의 제품 가치:

- 법적 저작자임을 자동 확정하는 것이 아니라, 과정기록과 발급 시점 무결성의 강한 증거를 남기는 것.
- 작가에게 팔 수 있는 가치는 “직접 쓴 걸 무조건 증명”이 아니라 “작업 과정과 제출 시점 무결성을 검증 가능하게 남김”.
- 확인서, 출고 패키지, 저작권 등록 준비 패키지, 권리/IP 점검이 한 흐름으로 묶여야 결제 명분이 생김.

최근 보강:

- 확인서 발급 전 과정기록 해시 체인 검문을 추가.
- 체인이 깨져 있으면 확인서 발급 차단.
