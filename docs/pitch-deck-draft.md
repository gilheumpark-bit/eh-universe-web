## 슬라이드 2 — Problem & Philosophy

### 제목
**"AI가 쓰면, 작가는 무엇인가?"**

### 본문

2026년 현재, AI 집필 시장 **Sudowrite $10M+ ARR** 달성.
그런데 작가 커뮤니티에서 반복되는 말:

> "AI로 썼으면 니 작품 아니잖아."
> "너무 잘 쓰는 AI 쓰면 내 글이 사라져."
> "완성도는 좋은데 내 거 같지 않음."

**주어가 사라졌다.** → 해결 = 주어를 돌려주기.

| 해줘 | 하라 |
|------|------|
| AI가 씁니다 | **작가가 AI로 씁니다** |
| 위임 = 권리 포기 | **통제 = 권리 유지** |
| 품질 = AI 출력 | **품질 = 작가 × AI 협업 결과** |

> **우리는 AI를 작가의 자리에 올리지 않습니다.**
> **작가의 손에 쥐여드립니다.**

### 시각 요소
- 좌: "AI가 쓴 소설" (Sudowrite 스크린샷)
- 우: 작가 손 + Loreguard 에디터 (작가 주어 은유)
- 하단 중앙 큰 글씨: **"같이 하세요."**

### 발표 스크립트 (45초)

> "Sudowrite는 이미 연매출 1천만 달러입니다. 그런데 작가 커뮤니티에선 계속 같은 말이 나옵니다. 'AI로 썼으면 니 작품 아니다.'
> 
> 문제는 **주어가 사라졌다**는 겁니다. AI가 주어가 되면 작가의 정체성·수익·성장 전부 증발합니다.
> 
> 저희 해법은 단순합니다. **주어를 작가에게 돌려줍니다.** AI는 작가의 자리가 아니라, 작가의 손에 있는 도구입니다."

## 슬라이드 3 — Solution (하라 전용 기능)

### 제목
**"해줘 도구는 할 수 없는 걸 우리는 한다"**

### 본문

Loreguard는 **제작 도구**입니다. Adobe · VS Code와 같은 카테고리.
해줘 도구에는 **필요 없지만**, 진짜 작가에게는 **필수**인 7가지:

| # | 기능 | 해줘 도구 | 하라 이유 |
|---|------|-----------|-----------|
| 1 | **Ctrl+K 팔레트** | ❌ | 전문가는 마우스 안 씀 |
| 2 | **Git 평행우주 브랜치** | ❌ | IF 전개 버전 관리 |
| 3 | **Rename Symbol** | ❌ | 캐릭터명 일괄 변경 |
| 4 | **Outline / Breadcrumbs / Minimap** | ❌ | 장편 네비게이션 |
| 5 | **Plugin Marketplace** | ❌ | 전문가별 확장 니즈 |
| 6 | **WorkProfiler** | ❌ | 작가별 문장 학습 |
| 7 | **특허 응답 제어** | ❌ | AI 통제권 확보 |

**+ 풀스택 파이프라인**: 세계관 → 집필 → 번역 → 출판 (4층 통합)
**+ DGX Spark 로컬 추론**: API $0 · 원고 서버 미전송

### 시각 요소
- 7개 기능 아이콘 격자 (3×3, 마지막 2칸은 "파이프라인" + "로컬 추론")
- 우리 열 전부 녹색, 해줘 열 전부 회색
- 하단 캐치: **"이게 전문가 IDE만 가질 수 있는 기능입니다."**

### 발표 스크립트 (60초)

> "Adobe에 있는 기능은 Canva에 없죠. 이유는 간단합니다. **제작자만 필요한 기능**이거든요.
> 
> 저희 Loreguard도 마찬가지입니다. Ctrl+K 팔레트, Git 브랜치, Rename Symbol, Plugin Marketplace — 이건 **해줘 도구엔 필요 없습니다.** 사용자가 통제 안 하니까요.
> 
> 하지만 진짜 작가 — 장편 50화 연재하는 작가, 캐릭터 20명 관리하는 작가, 1년 내내 한 세계관 쓰는 작가에겐 **필수**입니다.
> 
> 그리고 이걸 세계관부터 집필·번역·출판까지 **한 파이프라인으로 묶은 건 저희뿐**입니다. DGX Spark로 로컬 추론도 하니까 API 비용 0원, 원고는 서버 안 갑니다."

## 슬라이드 10 — Why Now (카테고리 타이밍)

### 제목
**"지금이 하라 카테고리를 창시할 마지막 기회"**

### 본문

5가지 조건 동시 충족:

1. **AI 모델 성능 충분** (Qwen 9B로 로컬 가능)
   → 1년 전 불가능. GPU 가격 · 모델 크기 · FP8 양자화가 지금 교차.

2. **웹소설 시장 폭발**
   → 한국 GSP 2.5조원. 일본 なろう 성숙. 중국 起点 수조원.
   → KR/JP/CN 각 시장 모두 **제작 도구 수요** 등장.

3. **작가의 AI 피로**
   → "AI가 써준다"는 이미 식상. 2025년 Sudowrite 붐 후 작가 커뮤니티 반작용 본격화.
   → **"내가 쓰고 싶다"**는 니즈가 명시적으로 표출되기 시작.

4. **"해줘" 시장 포화 — '하라' 시장 미개척**
   → 해줘 시장에만 10+ 경쟁사. 하라 시장은 **0**.
   → 지금이 **선점 윈도우**.

5. **특허 출원 타이밍 (AI 규제 대두 직전)**
   → 2026~2027년 AI 저작권 규제 본격화 예상.
   → 우리 특허는 규제 대응 인프라. **선출원 완료.**

### 한 줄 결론

> **"지금이 하라 카테고리를 창시할 마지막 기회입니다.**
> **6개월 후엔 후발 하라 플레이어가 등장할 겁니다."**

### 시각 요소
- 5개 조건 겹침 Venn diagram
- 중앙 교집합: "로어가드 창시 윈도우"
- 하단: "6~12개월 후 → 후발 진입" 경고 화살표

### 발표 스크립트 (45초)

> "왜 지금이냐? 5가지 조건이 처음으로 동시에 맞물렸습니다.
> 
> AI 모델이 **로컬 추론 가능한 수준**에 도달. 웹소설 시장이 한중일 각 **수조 원**. 작가들이 AI 피로 느끼기 시작. 해줘 시장은 **포화**. 특허 출원 가능한 타이밍.
