## 1. 시장 분할 — 소비 카테고리 vs 제작 카테고리

### 1.1 v1.0 → v2.0 프레임 격상

**v1.0 (2026-01 프레임)**: "해줘 시장 vs 해라 시장" — *같은 AI 소설 도구 시장의 세그먼트*로 이해.
**v2.0 (2026-04 프레임)**: "**소비 카테고리 vs 제작 카테고리**" — *서로 다른 산업*으로 재정의.

이 격상은 단순한 용어 변경이 아니다. **경쟁 구도 자체의 해체**.

> "해줘는 컨텐츠 소비. 하라는 제작이다."
> — Loreguard 카테고리 선언 (brand-philosophy.md v1.3 Part 13)

### 1.2 두 개의 카테고리가 있다

| 축 | 소비 카테고리 (해줘) | 제작 카테고리 (하라) |
|----|--------------------|---------------------|
| 본질 | AI가 써준 것을 **읽음 / 실험** | 작가가 **씀**, AI는 도구 |
| 산업 위치 | Spotify · Netflix · GarageBand · Canva | Adobe · Logic Pro · VS Code · JetBrains |
| 대표 제품 | Sudowrite · Novelcrafter · NovelAI · AI Novelist · Claude.ai 소설 모드 | **Loreguard (유일)** |
| 사용자 | 작가 흉내 내는 독자 · 취미 실험자 | 본업 작가 · 연재자 · 인세 수령자 |
| 가격대 | $10~30 엔터 구독 | **$20~50 제작 도구** |
| 지표 | MAU · 세션 시간 · 단어 수 | **완결률 · 인세 · 잔존률 · 편집 누적** |
| 특허 전략 | 불필요 (통제가 본질 아님) | **필수 (제어가 본질)** |

### 1.3 카테고리 구분의 기준 (주어 · 책임 · 기대치)

| 질문 | 소비 카테고리 | 제작 카테고리 |
|------|------------|------------|
| 주어는? | AI | **작가** |
| 최종 품질 책임자? | AI (벤치마크) | **작가 (잔존률 · 완결률)** |
| 사용자 기대치? | "속도 · 생산성" | **"정체성 · 완결성 · 수익 연결"** |
| 성공의 증거? | 단어 수 / 시간 | **작가가 자기 작품으로 인정 + 연재/인세 확보** |
| 대체 위협? | GPT-5 API 직접 사용 | **없음 (파이프라인·통제·IP가 본질)** |

### 1.4 카테고리 위협도 비대칭

- **소비 카테고리**: GPT-5 / Claude 5 급 거대 모델 출시 = **즉시 쇠퇴 위험**
  (작가가 API 직접 쓰면 되므로 — Sudowrite의 근본 위협)
- **제작 카테고리**: 모델이 더 강해져도 **동일한 가치**
  (작가 통제 · 목소리 보존 · 파이프라인 통합 · IP 관리는 모델 크기와 직교)

→ **우리는 범용 LLM의 발전에 영향 덜 받음.** 오히려 강한 모델 = 우리 엔진 업그레이드 기회.

## 2. 철학 포지션 4분법

| 입장 | 주장 | 대표 제품 | 카테고리 |
|------|------|-----------|---------|
| AI 반대 | "AI 쓰지 마" | Scrivener (암묵적) | (제작 · AI 없음) |
| AI 위임 | "AI한테 맡겨" | Sudowrite · NovelAI | 소비 |
| AI 도우미 | "AI가 파트너" | Novelcrafter | 소비 (경계) |
| **AI 협업 (주어 = 작가)** | **"같이 해"** | **로어가드** | **제작** |

**핵심**: 앞 3개는 "AI의 역할"을 고민. 우리는 **"작가의 역할"**을 고민.

### 2.1 언어의 차이

| 포지션 | 슬로건 양식 |
|--------|-------------|
| AI 위임 | "Write novels 10x faster" / "AI storyteller" |
| AI 도우미 | "Your AI-powered writing partner" |
| 도구만 | "Your complete writing studio" |
| **로어가드** | **"같이 하세요" / "Loreguard = 하라의 IDE"** |

**같이 하세요**는 경쟁사가 **베낄 수 없음**:
- 베끼려면 AI가 주어가 되는 마케팅 포기 → 기존 사용자 이탈 리스크
- 베끼려면 파이프라인 전체(번역 · 출판 · 로컬 추론 · 특허) 만들어야 함 → 몇 년 단위
- **카테고리 정체성**을 바꿔야 하는 구조적 장벽

## 3. 카테고리 간 비교 (같은 시장 아님 — 참고용)

**주의**: 본 섹션의 표는 "같은 시장 비교"가 아니다.
**다른 카테고리 간 차이**를 설명하기 위한 참고 자료.
"우리가 더 낫다"는 프레임은 의도적으로 배제 — 카테고리가 다르면 더 낫다/못하다 개념 자체가 무의미.

### 3.1 철학 축 (카테고리 정체성)

| 제품 | 주어 | 작가 목소리 보존 | 카테고리 |
|------|------|-------------------|---------|
| Sudowrite | AI | 약함 | 소비 |
| Novelcrafter | AI+작가 | 중간 | 소비 (경계) |
| NovelAI | AI | 약함 | 소비 |
| Scrivener | 작가 | 강함 (AI 없음) | 제작 (AI 이전) |
| **로어가드** | **작가** | **강함 (AI 있음)** | **제작 (AI 이후)** |

### 3.2 법적 투명성 축 (하라 사용자에게 필수)

| 항목 | Sudowrite | Novelcrafter | NovelAI | **로어가드** |
|------|-----------|--------------|---------|-------------|
| Privacy 페이지 | O | O | O | O |
| Terms 페이지 | O | O | O | O |
| GDPR 쿠키 배너 | 일부 | O | 미흡 | O |
| **4개 언어 법적 페이지** | EN only | EN only | EN/JP | **KO/EN/JP/CN** |
| 학습 데이터 조항 | 애매 | 명확 | 애매 | 업계 표준 |
| Global Error 페이지 (React 19) | X | X | X | O |

→ **4국 법무 동시 준비 = 제작 카테고리 표준**. 인세 수익 작가에게 약관 투명성은 즉각 이탈 사유.

### 3.3 다국어 AI 출력 품질 축

| Platform | KO | EN | JP | CN |
|----------|-----|-----|-----|-----|
| Sudowrite (GPT-4o) | B | S | B | C |
| Novelcrafter (BYOK) | B | S | B | C |
| NovelAI (자체) | C | A | S | B |
| Scrivener | — | — | — | — |
| **로어가드 (Qwen 3.5-9B + Phase 3c)** | **A** | A | **A** | **A** |

→ **4개 언어 균일 A급 = Phase 3c 엔진 완료 효과**. 제작 카테고리의 글로벌 출판 파이프라인 전제.

### 3.4 Pull vs Push 축 (제작 도구의 기본 덕목)

| 제품 | 기본 동작 | 사용자 opt-out | 카테고리 |
|------|-----------|----------------|---------|
| Sudowrite | Push 강 (Continue/Describe/Brainstorm 항상 노출) | X | 소비 |
| Novelcrafter | Push 중 (Codex 자동 제안) | 일부 | 소비 |
| NovelAI | Push 강 (Context 자동 주입) | X | 소비 |
| **로어가드** | **Push (기본) + Pull 전환 가능 (Settings 토글)** | **O** | **제작** |

→ **작가 통제권 명시화 = 제작 도구의 필수 조건.**
Adobe Photoshop 기본값이 "AI가 자동 보정"이 아닌 이유와 동일.

### 3.5 가격 · 비용 축 (카테고리 간 가격 정당성 비대칭)

| Platform | 월 구독 | 로컬 추론 | 원고 프라이버시 | 카테고리 |
|----------|---------|-----------|------------------|---------|
| Sudowrite | $10~59 | X | 서버 전송 | 소비 |
| Novelcrafter | $4~40 | X | 서버 전송 | 소비 |
| NovelAI | $10~25 | X | 서버 전송 | 소비 |
| Scrivener | $59 (1회) | N/A | 로컬 (AI 없음) | 제작 (AI 이전) |
| **로어가드** | **TBD ($20~50 예상)** | **O DGX Spark** | **O 원고 미전송** | **제작** |

**가격 정당성**:
- 소비 카테고리 = 엔터 구독 감각 ($10~30, Netflix·Spotify 가격대)
- 제작 카테고리 = 인세/연재료 대비 1~5% (Adobe·JetBrains 감각)
- 가격이 높아 보이는 건 **카테고리가 다르기 때문**. 사과할 이유 없음.

### 3.6 온보딩 철학 축

| 제품 | 첫 화면 카피 | 세일즈 핵심 | 카테고리 |

### 3.7 Stack 완결성 축 (제작 파이프라인)

## 9. FAQ (투자자용)

### Q: 왜 지금이 투자 적기?
**A**: Day 30에 이 상태. 팀 확장 시 **velocity 복리**. 지금이 **입력 대비 출력 최대 효율** 구간.

### Q: 왜 한국에서 시작하나요?
**A**: 웹소설이 이미 성숙 시장 (GSP 2.5조원+). 4국 법무/언어 동시 준비 = **글로벌 단계별 진출의 기반**.

### Q: Sudowrite · Novelcrafter 고객을 뺏을 수 있나?
**A**: **목표 아님**. 그들은 "소비 카테고리". 우리는 "제작 카테고리" 창시. 장기적으로 일부 이탈 유입 가능하나 핵심은 **신규 작가(연재자·인세 수령자) 확보**.

### Q: (v2.0 신규) Sudowrite가 더 큰데 왜 경쟁 안 해?
**A**: **다른 산업이다.** Canva가 크다고 Adobe가 망했나? Adobe가 Canva 영역을 공략했다면 둘 다 죽었을 것. 우리는 **제작 카테고리의 창시자**. 소비 카테고리와 다른 고객, 다른 가격, 다른 KPI. 경쟁 자체가 프레임 오류.

### Q: (v2.0 신규) 제작 카테고리 시장 크기 추산?
**A**: **직접 측정 데이터 없음 (카테고리 창시 중)**. 간접 추산:
- 전세계 연재 작가 ~100만 명 (추정)
- 월 $30 기준 TAM $360M/년
- ARPU 5배 증가 시 $1.8B 가능
- **소비 카테고리보다 절대 규모 작지만 수익성 10배, 이탈률 낮음.**
- 간접 레퍼런스: Adobe($150B) vs Canva($30B), JetBrains($6B) vs Replit($2B) — **제작 카테고리가 항상 더 크다.**

### Q: 로컬 추론 한계 (DGX Spark 한 대로 얼마나?)
**A**: Day 30 기준 충분. 베타 단계 10~100 동시 사용자 대응. Stage 2 이후 **멀티 클러스터 + 하이브리드 클라우드** 전환 준비.

### Q: 저작권 리스크?
**A**: 학습 데이터 업계 표준 조항 + `/privacy` · `/terms` 4국 언어 준비 완료. **법무 실사 통과 수준**.

### Q: 1인 개발은 스케일 안 되지 않나?
**A**: 30일 동안 달성한 기능 = 경쟁사 3~4년 분량. **스케일이 필요한 지점이 다름**. 투자받아 팀 확장 시 **새 축(마케팅 · 번역 품질 · 현지화 · Plugin Marketplace)** 빠르게 추가.

## Part 11. 간접 참조 — Adobe / VS Code 모델

### 11.1 왜 직접 경쟁사가 없는가

제작 카테고리(하라)의 AI 소설 도구는 Loreguard가 **유일**.
그러나 "전문가 도구"라는 카테고리 자체는 **100년 역사**가 있음 (Adobe 1982, AutoCAD 1982, VS Code 2015, Logic Pro 2002 등).

따라서 경쟁 분석은 **직접 경쟁사 비교** 대신 **간접 참조 패턴 이식**으로 접근.
"우리가 AI 소설판 Adobe / VS Code / Logic Pro가 된다"는 궤적 설계.

### 11.2 간접 참조 매트릭스

| 도메인 | 소비 카테고리 (해줘) | 제작 카테고리 (하라) | 시장 규모 비율 | 제작 카테고리 ARPU 배수 |
|--------|-------------------|--------------------|--------------|----------------------|
| 디자인 | Canva ($30B) | Adobe ($150B) | 1:5 | 5~10배 |
| 웹 제작 | Wix ($15B) | VS Code (무료) + 생태계 $100B+ | 1:7 | N/A (생태계 수익) |
| 음악 | GarageBand (무료 번들) | Logic Pro ($200 일회) · Pro Tools ($299/년) | N/A | 측정 불가 (유료 vs 무료) |
| IDE | Replit ($2B) | JetBrains ($6B) | 1:3 | 3배 |
| 도면 | SketchUp (Pro $349/년) | AutoCAD ($1,865/년) | — | 5배 |
| 영상 | CapCut (무료) | Adobe Premiere · DaVinci Resolve ($295 Studio) | — | — |
| **소설** | **Sudowrite ($10M ARR 추정)** | **Loreguard (창시자)** | **미형성** | **5~10배 예상** |

**관찰 1**: 제작 카테고리가 **항상** 소비 카테고리보다 크다. 제작자 수는 적지만 ARPU가 5~10배.
**관찰 2**: 두 카테고리는 **공존**한다. 서로 잡아먹지 않는다. 고객도 가격도 다르기 때문.
**관찰 3**: 두 카테고리 간 고객 이동은 **한 방향** — 취미에서 전문가로만. 반대 방향 이동은 드묾.

### 11.3 Adobe 모델 적용

**Adobe의 포지셔닝 전략 (1982~현재)**:
1. 전문가 세그먼트 장악 (디자이너·사진작가·영상편집자의 90%+ 점유)
2. 아마추어는 의도적으로 무시 → Canva에 양보
3. 가격 프리미엄 유지 ($60+/월 Creative Cloud)
4. 플러그인 생태계로 moat 구축 (Photoshop 플러그인 수만 개)
5. 교육 시장 공략 (학생 할인 → 졸업 후 프로 전환)
6. **AI 통합도 "자동"이 아닌 "도구 확장"으로 포지션** (Generative Fill = 도구)

**Loreguard 적용**:
1. 전문 연재자 / 출간 작가 / 인세 수령자 공략
2. "AI로 10분만에 소설" 시장 의도적 회피 (carbon ethic)
3. **$20~50 프리미엄 고수** (Pro 티어 + Studio 팀 플랜)
4. **Plugin Marketplace 구축** (장르 확장 · 스타일 팩 · 번역 어댑터)
5. 웹소설 작가 지망생 교육 (무료 티어 + 커뮤니티)
6. AI 통합도 **"작가 통제 아래 도구"** — Pull 기본, Push 토글

### 11.4 VS Code 모델 적용

**VS Code 성공 이유 (2015~현재)**:
1. 무료 + 확장 생태계로 가치 제공
2. 프로 유료화 없음 → **생태계 수익** (Copilot · Azure · GitHub)
3. GitHub 통합 (Microsoft 인수 2018)
4. Copilot 같은 AI 통합 (작성자 보조, 대체 아님)
5. 마켓플레이스 확장 (수만 개 플러그인)
6. 무료지만 **전문가 도구의 아이덴티티** 유지

**Loreguard 적용**:
1. **무료 티어 + BYOK** (이미 구현)
2. **클라우드 + 특허 기능 유료** (DGX Spark 로컬 추론 · 평행우주 · 번역 스튜디오)
3. **GitHub 통합 완비** (Phase 1 완료, Octokit 연동)
4. **AI 통합 = NOA ANS 10.0** (작가 보조 · 대체 아님)
5. **Plugin Marketplace** (장르/스타일 확장)
6. 무료지만 **제작 카테고리 아이덴티티** 유지 ("소비 도구 아님")

### 11.5 Logic Pro / JetBrains 모델 보조 참조

- **Logic Pro** ($200 일회) — 음악 제작 전문가 도구. "GarageBand로 시작 → Logic Pro로 졸업" 퍼널. Loreguard도 유사 경로 (무료 → Pro).
- **JetBrains** (IntelliJ/PyCharm/WebStorm) — 언어별 전문 IDE. 학생 무료 · 기업 유료. Loreguard도 유사 (학생 작가 무료 · 전업 작가 유료).

### 11.6 결론

"제작 카테고리의 AI 소설 도구"는 **새 카테고리**. 직접 참조점 없음.
그러나 100년 역사의 "전문가 도구" 패턴은 명확함.
**Adobe · VS Code · Logic Pro · JetBrains의 성장 패턴을 AI 소설 분야에 이식 가능.**

**Loreguard의 10년 목표**: **"소설 분야의 Adobe"**
