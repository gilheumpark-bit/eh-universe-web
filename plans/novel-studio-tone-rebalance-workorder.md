# NOA Studio — 작업장 느낌 완화 / 서사 몰입형 톤 재정렬 작업지시서

> 생성일: 2026-04-07  
> 기준 레포: `eh-universe-web`  
> 대상 경로: `/studio`  
> 목적: 기능은 유지하되, 사용자가 `업무용 작업장`보다 `EH 세계관 안의 집필 스튜디오`로 체감하도록 UI 톤과 정보 위계를 재설계한다.

---

## 0. 이 문서가 필요한 이유

현재 `eh-universe-web`는 구조상 매우 강한 제품이다.  
홈 포털, 소설 스튜디오, 코드 스튜디오, 번역 스튜디오, 네트워크가 하나의 디자인 시스템 안에 묶여 있다.

문제는 `/studio`가 기능적으로는 소설 창작 스튜디오인데, 사용자 체감상 일부 구간이 `작업 툴`, `운영 콘솔`, `워크벤치`처럼 읽힌다는 점이다.

이번 작업의 핵심은 새 기능을 많이 넣는 것이 아니다.  
기존 기능과 데이터 구조를 최대한 유지한 채, 화면의 말투와 구조, 강조 순서, 질감을 조정해서:

- `코드 스튜디오 같은 도구성`은 줄이고
- `EH 세계관 기반 창작 공간` 느낌은 올리고
- `집중감 있는 집필 경험`을 강화하는 것

에 있다.

## 1. 전체 분석 요약

### 1-1. 현재 레포의 강점

다음 요소는 이미 아주 좋다. 이번 작업에서도 유지해야 한다.

- 전역 테마가 차가운 SF 콘솔보다 `archive / editorial base`에 가깝다.
  - `src/app/globals.css`에 따뜻한 배경, 앰버 포인트, 세리프 조합이 이미 잡혀 있다.
- 루트 폰트 조합이 좋다.
  - `src/app/layout.tsx`에서 `Cormorant Garamond`, `Noto Serif KR`, `IBM Plex Sans`, `JetBrains Mono`를 같이 쓰고 있어 문서성과 도구성을 동시에 조절할 수 있다.
- 홈(`/`)은 `포털 + 세계관 허브` 톤이 강해서 브랜드 방향이 선명하다.
- `/studio`는 이미 탭 기반 구조라서, 사용자에게 기능을 이해시키기 쉽다.
- 세계관/캐릭터/룰북/집필/문체/원고/비주얼/히스토리/문서 등 서사 제작 흐름이 탄탄하다.

### 1-2. 현재 `/studio`가 작업장처럼 느껴지는 원인

코드를 기준으로 보면 아래 요소가 누적되면서 툴 감도가 올라간다.

1. 일부 직접적인 카피
- `src/components/studio/StudioSidebar.tsx`
  - `Writing Workbench`
- `ARCHITECTURE.md`
  - `AI-assisted narrative workbench`

2. 상단/사이드/패널에 모노스페이스 대문자 라벨이 과하게 많음
- `NOA STUDIO`
- `Live Feed`
- `Reference`
- `SCORE`
- `ANS`
- 상태 뱃지와 HUD성 표현들

3. 집필 탭의 정보 배치가 편집실보다 운영 패널처럼 보이는 구간이 있음
- `src/components/studio/tabs/WritingTabInline.tsx`
- `src/components/studio/tabs/RightChatPanel.tsx`

4. 코드 스튜디오에서 써도 어색하지 않은 시각 어휘가 일부 섞여 있음
- 강한 분할선
- 작은 시스템 배지 다중 노출
- 과한 보조 정보 동시 표시
- 실시간 패널 / 피드 / 상태 HUD 느낌

### 1-3. 중요한 판단

이번 단계에서는 `멀티윈도우`, `데스크톱형 창 관리자`, `하단 실행 도크`를 기본 UX로 도입하지 않는다.

이유:

- 현재 `/studio`는 탭 기반 구조로 이미 깊게 엮여 있다.
- 지금 당장 창 기반 UX로 넘어가면 사용자가 `소설 스튜디오`보다 `작업용 운영툴`로 더 읽을 가능성이 높다.
- “작업장 느낌 완화”라는 목표와 정면 충돌할 수 있다.

즉, **1차 목표는 구조 혁신이 아니라 톤 재정렬**이다.

## 2. 최종 목표

사용자가 `/studio`에 들어왔을 때 첫 인상이 아래처럼 바뀌어야 한다.

기존 인상:
- 기능이 많다
- 뭔가 전문 도구 같다
- 설정하고 관리하고 조작하는 느낌이 강하다

목표 인상:
- 세계관 안에서 글을 쓰는 공간 같다
- 정보는 많지만 차갑지 않다
- 작업보다는 집필과 구상에 초점이 있다
- AI는 앞에 나서지 않고 조용히 돕는다

한 줄로 정리하면:

**“IDE형 서사 워크벤치”가 아니라 “EH 세계관형 집필실”로 보이게 만들 것.**

## 3. 이번 작업의 범위

### 포함

- `/studio` 셸의 카피, 시각 위계, 간격, 표면 질감 조정
- 사이드바, 상단 헤더, 집필 탭, 우측 보조 패널, 모바일 탭바의 톤 정리
- 문장/라벨/버튼 텍스트 재작성
- 시맨틱 토큰 재사용 범위 내에서 스타일 조정
- 필요 시 번역 키 추가
- 문서 표현 정리

### 제외

- `StoryConfig` 스키마 변경
- 세션 저장 방식 변경
- AI API 구조 변경
- 탭 구조 자체 해체
- `/code-studio` 설계 규칙을 `/studio`에 가져오는 작업
- 대규모 새 엔진 기능 추가
- 멀티윈도우/도크 기반 완전 신설

## 4. 절대 원칙

1. **기능보다 인상 조정이 우선**
- 사용자에게 보이는 말투, 정보 밀도, 강조 순서부터 바꾼다.

2. **코드 스튜디오화 금지**
- `/studio`에 `/code-studio`의 패널 문화나 개발자용 UI 문법을 이식하지 않는다.

3. **EH 포털 톤 유지**
- 홈과 아카이브에서 느껴지는 `서사 아카이브`, `문서실`, `기록 보관소`의 질감을 유지한다.

4. **AI는 전면 주인공이 아니다**
- AI가 시스템을 지휘하는 느낌보다, 집필을 조용히 돕는 동반자처럼 보여야 한다.

5. **모든 것을 화려하게 바꾸지 않는다**
- 이번 작업은 감성 과장이 아니라 밀도 조정이다.

## 5. 디자인 방향

### 5-1. 유지할 것

- 따뜻한 다크/라이트 테마
- 앰버 중심 포인트 컬러
- 세리프 제목 + 산세리프 본문
- 종이/원고/아카이브를 연상시키는 질감
- 기존 탭 기반 정보 구조

### 5-2. 줄일 것

- 대문자 모노 라벨 남발
- `HUD`, `Live Feed`, `Workbench`, `Score` 같은 운영 패널 어휘
- 분절이 과한 패널 테두리
- 작은 상태 정보의 과밀 노출
- 기술 제품처럼 보이는 툴바 과잉

### 5-3. 강화할 것

- 작품 제목, 에피소드, 현재 집필 맥락
- 원고 본문 집중도
- 참고자료의 “보조성”
- 세계관과 문체가 주는 분위기
- 한 화면 안에서도 “읽기 쉬운 호흡”

## 6. 화면별 목표

## 6-1. 사이드바

### 현재 문제

- 프로젝트 선택, 새 에피소드, 탭 이동, 내보내기, 계정/동기화가 모두 강한 작업 메뉴처럼 보인다.
- 헤더의 `Writing Workbench`가 전체 인상을 바로 업무용 도구로 끌고 간다.

### 목표

사이드바는 `제어판`이 아니라 `집필실 입구`처럼 보여야 한다.

### 변경 지침

- `Writing Workbench` 문구 제거
- 상단 브랜딩은 `NOA Studio` 또는 `Novel Studio` 중심으로 단순화
- 프로젝트 선택 영역은 `폴더/콘솔`보다 `작품 보관함`처럼 보이게 정리
- 버튼은 기능적으로 유지하되, 라벨을 덜 기계적으로 수정
- 탭 목록은 “메뉴”보다 “집필 흐름”처럼 읽히게 여백과 타이포를 조정
- 내보내기/동기화/계정은 하단 유틸 블록으로 분리

### 카피 방향 예시

- `Writing Workbench` → `Narrative Studio` 또는 `Story Room`
- `Active Project` → `현재 작품`
- `New Episode` → `새 회차 시작`
- `Guided Mode / Free Mode` → `안내형 / 자유형`

---

## 6-2. 상단 헤더

### 현재 문제

- 상단에 시스템 제어 버튼과 상태 표시가 많아 “편집 도구 상단바” 느낌이 난다.

### 목표

상단 헤더는 “앱 제어 바”보다 “현재 집필 상태 요약선”에 가까워야 한다.

### 변경 지침

- 현재 작품 제목과 회차 맥락을 중앙 시선에 더 가깝게 둔다
- 저장 상태는 조용하게 유지
- `ANS` 뱃지는 기능은 남기되, 과도한 엔진 강조를 줄인다
- 검색/단축키/포커스모드 버튼은 2선급으로 낮춘다
- 아이콘 버튼 배경과 보더 대비를 낮춰 “장비 패널” 느낌을 줄인다

---

## 6-3. 집필 탭 본문

### 현재 문제

- 집필 모드 전환 버튼과 보조 패널이 강하게 드러나서 “글쓰기 공간”보다 “기능 조작 공간”처럼 느껴질 수 있다.

### 목표

집필 탭의 주인공은 항상 본문이다.

### 변경 지침

- 본문 컬럼을 시각적으로 더 조용하고 넓게 보이게 한다
- 상단 모드 전환 바는 유지하되, 컨트롤 패널보다 탭형 전환처럼 다듬는다
- `ContinuityGraph`, `Applied Settings`, 엔진 보조 정보는 본문보다 앞서 보이지 않게 한다
- 본문 영역은 종이/원고/원문 열람에 가까운 분위기로 정리한다
- `AI 생성`, `직접 편집`, `3단 캔버스`, `자동 리파인`, `고급`은 유지하되 카피 톤을 매끈하게 정리한다

### 주의

- 집필 기능은 그대로 둔다
- 현재 7:3 구조 자체는 유지 가능
- 다만 사용자 체감상 “오른쪽 패널이 주인공”처럼 보이지 않게 조절한다

---

## 6-4. 우측 보조 패널

### 현재 문제

- `RightChatPanel`이 `Live Feed`, `Reference`, `SCORE`, 파이프라인 정보 등으로 구성되며 운영형 어조가 섞여 있다.

### 목표

이 패널은 `라이브 운영 콘솔`이 아니라 `집필 조수 노트`처럼 보여야 한다.

### 변경 지침

- `Live Feed` 같은 표현 제거
- `Reference`는 `참고 메모`, `현재 장면 참고`, `등장인물 메모`처럼 더 서사적인 이름으로 전환
- HFCP 점수/판정은 필요 시 축약 또는 접힘 처리
- 파이프라인 정보는 기본 접힘 상태 또는 비강조 영역으로 내린다
- 메시지 카드 스타일은 채팅앱보다 메모/주석 카드에 가깝게 조정
- `Assistant Ready` 같은 문구도 덜 시스템적으로 바꾼다

### 카피 방향 예시

- `Reference` → `참고 메모`
- `Live Feed` → `대화 기록`
- `Assistant Ready` → `필요할 때 바로 도와줄 수 있어요`
- `SCORE` 직접 노출 → 기본 숨김 또는 서브 텍스트화

---

## 6-5. 모바일 하단 탭바

### 현재 문제

- 모바일 탭바는 품질은 좋지만, 일부 구간이 앱 쉘 느낌이 강하다.

### 목표

모바일에서도 `작업 앱`이 아니라 `창작 스튜디오`처럼 읽혀야 한다.

### 변경 지침

- 활성 효과를 조금 더 절제한다
- 빛나는 인디케이터나 과한 글로우는 완화한다
- 라벨 어휘는 기능보다 맥락 중심으로 조정할 수 있다
- 하단 바 전체를 기기 UI처럼 보이게 하기보다 작품 흐름 네비처럼 조정한다

## 9. 성공 기준

아래를 만족하면 성공이다.

1. 첫 화면에서 `도구`보다 `집필 공간`처럼 느껴진다
2. 사용자가 현재 작품과 회차에 더 빨리 시선을 둔다
3. AI 보조 패널이 시끄럽지 않고 자연스럽다
4. 기존 기능은 그대로 동작한다
5. 홈 포털과 소설 스튜디오의 브랜드 결이 더 잘 이어진다
