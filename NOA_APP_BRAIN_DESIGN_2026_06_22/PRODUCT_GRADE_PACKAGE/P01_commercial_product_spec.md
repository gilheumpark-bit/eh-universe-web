## 1. 제품명 후보

## 2. 제품 정의

NOA Studio Brain은 EH Universe Web 안에서 작동하는 AI 창작 관제층이다.

제품 기능은 세 범주로 나뉜다.

| 범주 | 설명 |
|---|---|
| Expert Assist | 탭별 전문가 조언 |
| Safe Apply | AI 제안 적용 전 위험 판단 |
| Work Memory | 변경 이력, 근거, 복구 힌트 기록 |

## 3. 사용자 역할

| 역할 | 주요 작업 | 필요한 기능 |
|---|---|---|
| Writer | 원고 작성, 장면 수정 | 집필 보조, 문체 유지, 적용 미리보기 |
| Worldbuilder | 세계관/설정 관리 | 모순 점검, 규칙 확장성, 설정 연결 |
| Character Designer | 캐릭터 설계 | 심리, 말투, 관계, 성장선 관리 |
| Editor/PD | 품질 관리 | 수정 요청, 변경 이력, 출고 전 점검 |
| Studio Lead | 팀 운영 | 프로젝트 상태, 위험 작업, 권한 관리 |
| Translator | 현지화 | 용어집, 말투, 문화 리스크 점검 |

## 4. 핵심 기능

### 4.1 탭별 전문가

각 탭은 동일한 챗봇을 공유하지 않는다.  
현재 탭, 작업 종류, 대상 데이터에 따라 전문가 모드를 전환한다.

```text
world -> 세계관 시스템 설계자
characters -> 캐릭터 심리 디렉터
plot/scene -> 서사 구조 설계자
writing -> 집필 파트너
revision -> 퇴고 외과의
translate -> 현지화 편집자
export -> 출고/권리 관리자
```

### 4.2 안전 적용

AI 제안은 바로 원고에 박히지 않는다.  
위험도에 따라 아래 중 하나가 선택된다.

| 상태 | 사용자 체감 |
|---|---|
| 적용 가능 | 작은 수정은 자연스럽게 반영 |
| 미리보기 | 바뀔 부분을 먼저 보여줌 |
| 나눠서 적용 | 큰 변경을 작은 단위로 분리 |
| 적용 보류 | 맥락이 부족하면 질문 또는 초안만 제시 |
| 작품 보호 | 원고/설정 손상 위험이 큰 작업은 중지 |

### 4.3 작업 영수증

중요 작업마다 아래 기록을 남긴다.

```text
무엇을 바꿨는가
왜 바꿨는가
어떤 탭 전문가가 판단했는가
어떤 위험을 감지했는가
되돌릴 수 있는가
```

### 4.4 출고 전 점검

출고/내보내기 단계에서는 단순 export가 아니라 다음을 확인한다.

| 점검 | 목적 |
|---|---|
| 설정 충돌 | 공식 설정 붕괴 방지 |
| 캐릭터 말투 | 캐릭터성 유지 |
| 회차 연결 | 누락/중복/단절 방지 |
| 권리/출처 메모 | 외부 사용 리스크 관리 |
| 변경 이력 | 팀 검토 및 복구 |

## 5. 주요 사용자 흐름

### Flow A. 세계관 설정 추가

```text
사용자: 새 국가/조직/규칙 추가
NOA: world 전문가 선택
NOA: 기존 설정과 충돌 점검
NOA: 적용 가능/미리보기/보류 결정
NOA: 설정 변경 영수증 생성
```

### Flow B. AI 원고 수정 적용

```text
사용자: 문단 개선 요청
NOA: writing 전문가 선택
NOA: 문체, 캐릭터 말투, 장면 목적 확인
NOA: 작은 수정은 적용 가능
NOA: 큰 구조 변경은 미리보기 또는 나눠서 적용
NOA: 변경 전후 비교와 복구 힌트 기록
```

### Flow C. 팀 출고 준비

```text
사용자: 출고 패키지 생성
NOA: export 전문가 선택
NOA: 세계관/캐릭터/원고/권리 메모 점검
NOA: 고위험 항목은 보호 정지
NOA: 출고 체크리스트와 누락 목록 생성
```

## 6. 기능 범위

| 버전 | 포함 |
|---|---|
| MVP | 탭 전문가, 적용 미리보기, 작업 영수증 |
| Pro | 원고/설정 충돌 점검, 히스토리 비교, 복구 안내 |
| Studio | 팀 작업, 권한, 출고 전 점검, 프로젝트 리포트 |
| Enterprise | 온프레미스/전용 저장소/감사 로그/조직 정책 |

## 7. 제품에서 쓰지 말아야 할 표현

고객 UI에는 아래 표현을 직접 쓰지 않는다.

```text
NIB
threshold
policy kernel
agentic control
enforcement
D128
```

대체 표현:

| 내부어 | 제품어 |
|---|---|
| policy gate | 적용 전 확인 |
| enforcement | 작품 보호 |
| receipt | 작업 기록 |
| depth | 검토 수준 |
| rollback | 되돌림 |
| externality | 외부 저장/내보내기 영향 |
