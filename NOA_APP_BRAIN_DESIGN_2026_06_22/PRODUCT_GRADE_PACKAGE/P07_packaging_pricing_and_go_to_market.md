# P07. Packaging, Pricing, and Go-To-Market

## 1. 결론

NOA Studio Brain은 단순 AI 채팅 과금보다 "작품 보호와 팀 워크플로우"로 패키징하는 편이 낫다.

가격 숫자는 실제 시장 조사와 원가 계산 전에는 확정하면 안 된다.  
이 문서는 상품 구조와 과금 축만 제안한다.

## 2. 상품 패키지

| 패키지 | 대상 | 핵심 가치 |
|---|---|---|
| Solo | 개인 작가 | 탭별 조언, 안전 적용, 기본 이력 |
| Pro Creator | 연재 작가/소규모 팀 | 원고/설정 충돌 점검, 복구, 출고 체크 |
| Studio | 제작팀/PD | 팀 작업, 감사 로그, 프로젝트 리포트 |
| Enterprise/IP | IP 사업자 | 조직 정책, 전용 저장, 권한, 보안/감사 |

## 3. 기능 티어

| 기능 | Solo | Pro | Studio | Enterprise |
|---|---|---|---|---|
| 탭별 전문가 | 기본 | 고급 | 고급 | 맞춤 |
| 안전 적용 | 기본 | 고급 | 고급 | 정책 기반 |
| 작업 영수증 | 최근 기록 | 전체 기록 | 팀 감사 | 조직 감사 |
| 출고 점검 | 기본 | 고급 | 승인 흐름 | 맞춤 체크 |
| 복구 힌트 | 제한 | 전체 | 팀 기준 | 조직 기준 |
| 권한/팀 | 없음 | 제한 | 포함 | 고급 |
| 데이터 정책 | 기본 | 프로젝트 | 팀 | 전용/계약 |

## 4. 과금 축

권장 과금 축:

```text
프로젝트 수
팀 좌석 수
작업 영수증 보관 기간
출고 패키지 수
고급 전문가/검토 깊이
전용 저장/보안 옵션
```

주의:

```text
AI 호출량만 과금 축으로 삼으면 제품 가치가 낮아 보일 수 있다.
작품 보호, 이력, 출고 안정성 중심으로 묶는 편이 낫다.
```

## 5. 판매 메시지

개인 작가용:

```text
AI로 더 빨리 쓰되, 작품 설정과 캐릭터를 잃지 않게 합니다.
```

팀/스튜디오용:

```text
원고, 설정, 수정 이력을 한 흐름에서 관리하고 출고 전 위험을 줄입니다.
```

IP 사업자용:

```text
세계관과 캐릭터 자산을 변경 이력과 권리 메모까지 포함해 관리합니다.
```

## 6. 첫 판매 가능 형태

첫 유료화는 전체 자동화보다 아래 구성이 현실적이다.

```text
탭별 전문가 조언
AI 적용 미리보기
작업 영수증
출고 전 체크리스트
데모 프로젝트
```

이 다섯 가지가 실제로 보이면 구매자가 가치를 이해하기 쉽다.

## 7. 데모 데이터

판매 데모에는 아래 샘플이 필요하다.

| 샘플 | 목적 |
|---|---|
| 세계관 충돌 샘플 | world 전문가 가치 증명 |
| 캐릭터 말투 흔들림 샘플 | character/writing 조합 증명 |
| 대량 원고 수정 샘플 | preview/split 가치 증명 |
| 출고 누락 샘플 | export guard 가치 증명 |
| 복구 샘플 | receipt/recovery 가치 증명 |

## 8. 도입 전략

권장 순서:

```text
1. 내부 데모 프로젝트 제작
2. 작가 1인용 closed test
3. 소규모 팀 pilot
4. 유료 pilot
5. Studio/Enterprise 패키지 확장
```
