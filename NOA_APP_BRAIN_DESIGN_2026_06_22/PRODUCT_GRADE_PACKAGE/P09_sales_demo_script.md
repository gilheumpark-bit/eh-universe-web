# P09. Sales Demo Script

## 1. 결론

판매 데모는 기능 나열이 아니라 "왜 돈을 낼 만한가"를 보여줘야 한다.

권장 데모 메시지:

```text
AI가 글을 써주는 것보다 중요한 것은,
AI가 작품을 망치지 않게 관리하는 것입니다.
```

## 2. 5분 데모 구성

| 시간 | 장면 | 보여줄 가치 |
|---|---|---|
| 0:00-0:40 | 프로젝트 열기 | 세계관/캐릭터/원고가 한 곳에 있음 |
| 0:40-1:30 | writing 탭 AI 수정 | 탭 전문가가 문체와 말투를 고려 |
| 1:30-2:10 | Apply Preview | 바뀔 부분을 먼저 보여줌 |
| 2:10-2:50 | Work Receipt | 왜 바꿨는지 기록 |
| 2:50-3:40 | world 충돌 감지 | 설정 붕괴를 막음 |
| 3:40-4:30 | export guard | 출고 전 누락/권리 메모 확인 |
| 4:30-5:00 | 요약 | 작품 보호형 AI 관제 가치 |

## 3. 데모 시나리오

### Scene 1. 일반 AI와의 차이

설명:

```text
일반 AI는 문단을 다시 써줍니다.
NOA Studio Brain은 그 문단이 캐릭터 말투와 세계관에 맞는지도 같이 봅니다.
```

보여줄 동작:

```text
writing 탭에서 문단 개선 요청
NOA가 바로 전체 덮어쓰기 대신 미리보기 제공
```

### Scene 2. 작품 보호

설명:

```text
대량 변경은 작가에게 가장 위험한 순간입니다.
NOA는 한 번에 바꾸지 않고 장면 단위로 나눕니다.
```

보여줄 동작:

```text
원고 전체 톤 변경 요청
Decision = SPLIT
장면별 적용 선택지 표시
```

### Scene 3. 설정 충돌

설명:

```text
새 설정은 좋아 보여도 기존 세계관 규칙을 깨뜨릴 수 있습니다.
```

보여줄 동작:

```text
world 탭에서 새 국가/기술 설정 추가
기존 canon과 충돌 후보 표시
```

### Scene 4. 출고 보호

설명:

```text
출고는 단순 저장이 아니라 공식화에 가깝습니다.
NOA는 출고 전 누락과 권리 메모를 확인합니다.
```

보여줄 동작:

```text
export 실행
누락 항목, 권리 메모 필요 항목, 최종 체크 표시
```

## 4. 구매자별 강조점

| 구매자 | 강조 |
|---|---|
| 개인 작가 | 빠르게 쓰면서 설정을 잃지 않음 |
| PD/편집자 | 수정 이력과 근거가 남음 |
| 제작팀 | 팀 전체가 같은 canon을 봄 |
| IP 사업자 | 출고/권리/자산 관리가 쉬워짐 |

## 5. 데모 실패 조건

아래가 보이면 판매 설득력이 떨어진다.

| 실패 | 문제 |
|---|---|
| 일반 챗봇처럼만 보임 | 차별점 없음 |
| 탭별 응답이 비슷함 | 전문가 구조가 약함 |
| 변경 전후 비교가 없음 | 안전 적용 가치가 약함 |
| receipt가 없음 | 팀/상용 가치 약함 |
| export guard가 없음 | 출고 제품으로 약함 |
