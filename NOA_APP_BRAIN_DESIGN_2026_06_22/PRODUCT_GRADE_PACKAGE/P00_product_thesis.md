# P00. Product Thesis

## 1. 결론

NOA App Brain의 상용 제품 명제는 다음이다.

```text
AI 창작 보조를 "답변 생성기"에서
"작품 손상을 막는 창작 관제 시스템"으로 바꾼다.
```

일반 AI 도구는 문장을 만들고 아이디어를 낸다.  
판매 가능한 NOA 제품은 그 다음 문제를 해결해야 한다.

```text
AI가 만든 제안이
내 세계관, 캐릭터, 원고, 출고본을 망치지 않는가?
```

## 2. 고객 문제

| 고객군 | 핵심 문제 | 구매 이유 |
|---|---|---|
| 개인 작가 | 설정과 원고가 흩어짐 | AI 제안을 적용해도 작품이 무너지지 않게 함 |
| 웹소설/웹툰 제작팀 | 캐릭터, 회차, 설정 관리가 어려움 | 팀 단위 변경 이력과 출고 안정성 |
| IP 기획자 | 세계관과 캐릭터 자산화가 어려움 | IP bible, 설정 충돌, 확장성 관리 |
| 편집자/PD | 수정 지시와 실제 반영 추적이 어려움 | 변경 근거, 검토 기록, 복구 후보 제공 |
| 번역/현지화 담당 | 이름, 말투, 문화 맥락이 흔들림 | 현지화 일관성과 위험 표현 점검 |

## 3. 제품 한 줄 설명

```text
NOA App Brain은 창작 앱 안에서 탭별 전문가처럼 판단하고,
AI 적용, 저장, 내보내기, 대량 수정의 위험을 관제하는 작품 보호형 AI 운영층이다.
```

## 4. 차별점

| 일반 문서 편집기 | 일반 AI 챗봇 | NOA App Brain |
|---|---|---|
| 원고를 저장함 | 답변을 생성함 | 변경 위험과 작품 맥락을 판단함 |
| 탭별 전문성 약함 | 대화 맥락 중심 | 세계관/캐릭터/플롯/원고/출고별 전문가 라우팅 |
| 변경 이유 추적 제한 | 답변 이력 중심 | 적용 전 판단과 적용 후 영수증 |
| 실수 복구는 사용자 책임 | 복구 설계 없음 | 되돌림 힌트와 작업 기록 중심 |

## 5. 구매자가 돈을 내는 지점

고객은 "더 많은 AI 답변"에만 돈을 내지 않는다.  
상용 제품에서 돈을 내는 지점은 다음이다.

```text
1. 작품 설정을 잃지 않음
2. 원고 적용 사고를 줄임
3. 팀 작업의 변경 근거를 남김
4. 출고 전에 위험을 잡음
5. AI를 썼다는 불안감을 관리 가능한 절차로 바꿈
```

## 6. 제품 포지셔닝

권장 포지셔닝:

```text
AI Creative Control Tower
```

한국어 제품 표현:

```text
작품을 지키는 AI 창작 관제실
```

보조 표현:

```text
AI가 써주는 앱이 아니라,
AI가 망치지 않게 관리하는 앱.
```

## 7. 비목표

상용 설계에서 피해야 할 방향:

| 비목표 | 이유 |
|---|---|
| 만능 연구원 주장 | 실제 제품 검증 범위를 벗어남 |
| 자동 완성만 강조 | 경쟁력이 낮음 |
| 사용자 허락 없는 대량 적용 | 작품 손상과 신뢰 하락 위험 |
| 복잡한 내부 용어 노출 | 구매자 이해와 온보딩 저해 |
| 모든 장르를 같은 방식으로 처리 | 창작 품질 저하 |

## 8. 상용 성공 조건

NOA App Brain이 판매 가능한 제품이 되려면 아래 조건을 만족해야 한다.

| 조건 | 기준 |
|---|---|
| 명확한 고객 가치 | 작품 보호, 적용 안전, 이력 관리가 체감되어야 함 |
| 데모 가능성 | 5분 안에 위험 적용을 막고 안전 적용을 보여줘야 함 |
| 반복 품질 | 같은 입력에 대해 일관된 판단이 나와야 함 |
| 복구 가능성 | 적용 후 되돌림 또는 비교가 가능해야 함 |
| 설명 가능성 | 왜 보류/분할/미리보기인지 사용자가 이해해야 함 |
