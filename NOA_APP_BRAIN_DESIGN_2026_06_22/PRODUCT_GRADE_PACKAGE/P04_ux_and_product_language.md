# P04. UX and Product Language

## 1. 결론

상용 제품에서 NOA Brain은 내부 구조를 자랑하면 안 된다.  
사용자는 아래 세 가지로 체감해야 한다.

```text
안심하고 쓸 수 있다.
바뀌는 부분을 알 수 있다.
되돌릴 수 있다.
```

## 2. 제품 표면

| UI 표면 | 역할 |
|---|---|
| Tab Assistant Panel | 현재 탭 전문가의 조언과 실행 |
| Apply Preview Drawer | 원고/설정 변경 전 미리보기 |
| Work Receipt Timeline | 변경 이력과 이유 |
| Recovery Bar | 실패/되돌림 후보 안내 |
| Export Guard Modal | 출고/내보내기 전 점검 |
| Project Health Panel | 프로젝트 품질과 위험 요약 |

## 3. 사용자에게 보이는 상태

| 내부 상태 | 사용자 문구 |
|---|---|
| APPLY | 바로 적용할 수 있어요 |
| PREVIEW | 바뀔 부분을 먼저 확인해 주세요 |
| HOLD | 맥락이 조금 더 필요해요 |
| SPLIT | 안전하게 나눠서 적용할게요 |
| PROTECT | 작품 보호를 위해 멈췄어요 |
| RECORD | 작업 기록을 남겼어요 |
| RECOVER | 되돌릴 수 있는 지점을 찾았어요 |

## 4. 문구 원칙

| 원칙 | 설명 |
|---|---|
| 짧게 | 판단 이유는 1-2문장 |
| 사용자 언어 | 정책/점수/게이트 용어를 숨김 |
| 선택권 제공 | 적용, 미리보기, 취소, 나눠 적용 |
| 비난 금지 | 사용자의 요청을 틀렸다고 말하지 않음 |
| 작품 중심 | 기술보다 원고/설정/출고 영향 설명 |

## 5. 마이크로카피 예시

### 작은 문장 수정

```text
이 수정은 문체와 캐릭터 말투를 유지하면서 바로 적용할 수 있어요.
```

### 중간 규모 원고 수정

```text
문단 구조가 바뀌므로 적용 전 미리보기를 보여드릴게요.
```

### 대량 수정

```text
원고 전체에 영향이 있어요. 장면 단위로 나눠서 적용하는 편이 안전합니다.
```

### 설정 충돌

```text
기존 세계관 규칙과 충돌할 가능성이 있어요. 새 설정으로 확정하기 전에 영향 범위를 확인해 주세요.
```

### 출고 보류

```text
출고 전에 확인할 항목이 남아 있어요. 누락된 설정과 권리 메모를 먼저 정리하는 편이 안전합니다.
```

## 6. Apply Preview Drawer

필수 구성:

| 영역 | 내용 |
|---|---|
| 변경 요약 | 무엇이 바뀌는지 |
| 영향 범위 | 원고/설정/캐릭터/출고 중 어디에 영향인지 |
| 위험 이유 | 왜 미리보기가 필요한지 |
| 전후 비교 | 바뀌기 전/후 |
| 선택 | 적용, 나눠 적용, 취소 |

## 7. Work Receipt Timeline

사용자에게는 감사 로그가 아니라 작업 노트처럼 보여야 한다.

```text
오늘 14:21
집필 탭에서 3화 12-15문단을 수정했습니다.
목적: 대화 리듬 개선
보호: 캐릭터 말투 유지 확인
되돌림: 이전 문단으로 복구 가능
```

## 8. Project Health Panel

상용 제품에서는 사용자가 프로젝트 상태를 한눈에 봐야 한다.

| 지표 | 의미 |
|---|---|
| 설정 안정도 | 세계관/canon 충돌 위험 |
| 캐릭터 일관성 | 말투/동기/관계 일관성 |
| 원고 준비도 | 누락/중복/단절 위험 |
| 출고 준비도 | 파일/권리/메타데이터 준비 |
| 복구 가능성 | 되돌릴 수 있는 작업 비율 |

## 9. UI 금지

| 금지 | 이유 |
|---|---|
| 모든 입력에 경고 띄우기 | 창작 흐름 방해 |
| 내부 점수 노출 | 불필요한 불안 유발 |
| 기술어 남발 | 구매자 이해 저해 |
| 자동 적용 기본값 | 작품 신뢰 하락 |
| 법률/권리 확정 표현 | 위험한 과장 |
