# NOA App Brain Option 1 — Loreguard Absorption Plan

작성일: 2026-06-22
범위: 현재 Loreguard Studio 흐름에 얇게 흡수하는 1안 구현 기준

## 목표

1안은 별도 뇌 시스템을 새 화면으로 노출하지 않는다. 현재 로어가드의 탭, 노아 도크, 집필 제안 흐름에 탭별 전문가 지시와 적용 판단 정책을 끼워 넣는다.

핵심은 다음 세 가지다.

- 사용자는 그대로 세계관, 캐릭터, 시나리오, 씬시트, 연출, 집필, 퇴고, 번역, 출고를 쓴다.
- 내부에서는 현재 탭을 기준으로 노아의 역할, 참고 깊이, 적용 기준을 바꾼다.
- 큰 변경, 외부 저장, 출고 같은 경계는 바로 적용보다 미리보기나 분할을 우선한다.

## 활성 탭 기준

정본 탭은 `LoreguardShell`의 10단계다.

| 탭 | 내부 역할 | 기본 깊이 | 핵심 판단 |
|---|---|---:|---|
| project | 프로젝트 관제 | D8 | 목표, 권리/IP, 출고 준비를 흩어지지 않게 묶는다 |
| world | 세계관 관제 | D32 | 설정 충돌, 규칙 비용, 후속 확장성을 본다 |
| character | 캐릭터 관제 | D32 | 욕망, 결핍, 말투, 관계 압력을 본다 |
| plot | 시나리오 관제 | D32 | 사건 인과, 회차 추진력, 결말 잠금을 본다 |
| scene | 씬시트 관제 | D32 | 장면 목적, 공개/은닉 정보, 다음 연결을 본다 |
| direction | 연출 관제 | D32 | 텐션, 후킹, 감정 곡선, 문장 리듬을 본다 |
| writing | 집필 관제 | D64 | 작가 지시, 기준선, 원고 반영 위험을 본다 |
| revision | 퇴고 관제 | D32 | 문장 결함, 반복, 톤 흔들림, 직접 수정량을 본다 |
| translate | 현지화 관제 | D64 | 원문 보존, 문화 어색함, 용어 일관성을 본다 |
| export | 출고 관제 | D64 | 과정기록, 권리/IP 점검, 제출 묶음을 본다 |

구 Studio 탭 값은 정본 탭으로만 변환한다. Code Studio, Network, Archive 같은 구표면은 되살리지 않는다.

## 판단 상태

1안의 정책 상태는 화면 노출용 기능명이 아니라 내부 적용 기준이다.

| 상태 | 의미 | 기본 처리 |
|---|---|---|
| APPLY | 바로 적용 가능 | 작은 노아 제안, 되돌릴 수 있는 편집 |
| PREVIEW | 미리보기 필요 | 원고 대량 반영, 외부 저장, 출고 묶음 변경 |
| HOLD | 정보 부족 | 작가 지시가 짧거나 기준선이 빠진 경우 |
| SPLIT | 나눠서 처리 | 여러 탭 또는 큰 범위를 한 번에 바꾸는 경우 |
| PROTECT | 보호 우선 | 권리/IP, 개인정보, 되돌리기 어려운 변경 |
| RECORD | 기록만 | 분석, 탭 전환, 일반 상태 확인 |
| RECOVER | 복구 흐름 | 저장 실패, 동기화 실패, 체인 불일치 |

## 적용 위치

이번 1안에서 직접 연결하는 곳:

- `ChatCanvasDock`: 세계관, 캐릭터, 시나리오, 씬시트, 연출 노아 대화의 시스템 지시에 탭 전문가 지시를 추가한다.
- `useWritingChat`: 집필 전용 노아 대화에 집필 탭 전문가 지시를 추가한다.
- `buildAIWritePromptFromContextPack`: 집필 제안 프롬프트 앞에 집필 탭 전문가 지시를 추가한다.

이번 1안에서 순수 함수만 제공하고, UI 차단은 보류하는 곳:

- 저장
- 외부 저장
- 출고
- 대량 적용
- 복구

이 경계들은 `decideAppBrain`으로 먼저 판정할 수 있게 만들고, 실제 UI 흐름 차단은 다음 패스에서 붙인다.

## 공개 표면 원칙

- 사용자 화면에서는 새 시스템명을 드러내지 않는다.
- 노아는 작가가 리드하는 작업 도우미로 남긴다.
- 일반 타이핑은 끊지 않는다.
- 큰 변경만 미리보기, 분할, 보류를 요구한다.
- 판단 결과는 과정기록과 확인서의 근거가 될 수 있게 원문 전체보다 해시, 요약, 사유 중심으로 남긴다.

## 1안 완료 기준

- 정본 탭 레지스트리가 코드에 존재한다.
- 구 탭 id가 정본 탭으로 안정적으로 정규화된다.
- 탭별 노아 지시문이 활성 Loreguard 도크와 집필 제안 프롬프트에 붙는다.
- 적용 정책 순수 함수가 단위 테스트로 검증된다.
- 타입 체크, 관련 Jest, 사용자 노출 문구 검사를 통과한다.
