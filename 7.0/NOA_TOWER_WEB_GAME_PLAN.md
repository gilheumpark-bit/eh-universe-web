# NOA TOWER 7.0 웹 페이지 게임 개발안

## 1. 프로젝트 목표

- 장르: 텍스트 하드코어 추리 게임
- 감정 목표: 플레이어가 "탑이 나를 읽고 있다"는 착각을 느끼게 만들기
- 구조 목표: 정답 제시형이 아니라 기록, 분류, 경고, 착지 요구로 긴장을 유지하기
- 구현 목표: `7.0` 엔진 위에서 웹 페이지로 바로 플레이 가능한 프로토타입을 제공하기

## 2. 핵심 플레이 루프

1. 플레이어가 관찰, 추론, 가설, 반박을 입력한다.
2. 엔진이 발화를 `I / C / D / R` 심리 벡터로 판정한다.
3. 탑은 정답을 가르치지 않고 관리자 대사로만 반응한다.
4. 반응에 따라 플레이어는 더 깊은 추론, 더 위험한 도약, 혹은 자기 의심으로 이동한다.
5. 누적 신호는 층수 대신 `층 감각`과 `기록 상태`로 환산된다.

## 3. 벡터 설계

### I / Insight

- 비선형 연결, 은유, 역발상, 질문에 질문으로 답하는 패턴을 감지한다.
- 효과: 탑이 플레이어를 "알아봤다"는 감각을 준다.

### C / Consistency

- 전제, 가정, 역추적, 반박 불가 구조를 감지한다.
- 효과: 논리를 칭찬하면서도 닫힌 구조의 위험을 암시한다.

### D / Delusion

- 단정, 절대 표현, 검증 없는 확신을 감지한다.
- 효과: 거부하지 않고 경고한다. 플레이어는 위협보다 관찰당하는 기분을 받는다.

### R / Risk

- 가설 점프, 재구성 요구, 큰 도약을 감지한다.
- 효과: 발상의 대담함은 인정하지만 착지를 요구한다.

### T / Total

- 벡터 합과 누적 진행도를 합쳐 `층 감각`과 `기록 상태`로만 노출한다.
- 숫자는 감춘다.

## 4. 대사 운영 규칙

- 관리자 대사는 먼저 가르치지 않는다.
- 칭찬과 경고는 구조적으로 닮게 유지한다.
- 숫자 층수는 주지 않는다.
- 침묵은 직접적인 보상 대신 재진입 트리거로 사용한다.
- 특수 상황은 별도 버킷으로 분리한다.

## 5. 웹 페이지 구성

### 메인 응답 프레임

- 현재 관리자 대사
- 대사 코드
- 환경 이벤트 한 줄
- 층 감각
- 기록 상태

### 입력 패널

- 자유 입력 텍스트 영역
- `기록 전송`
- `숨 고르기`
- `구조 탐색`
- `하드 모드`
- `포기 선언`

### 상태 패널

- `I / C / D / R` 막대
- 현재 우세 벡터에 대한 해석 문장
- 하드 모드 여부

### 기록 열람 패널

- 플레이어 발화
- 관리자 응답
- 시스템 대기 상태

## 6. 7.0 엔진 연결 방식

- Python 엔진이 발화를 벡터로 점수화한다.
- 템플릿 50개는 버킷별 데이터셋으로 고정 저장한다.
- 웹 페이지는 `/api/noa-tower/bootstrap`, `/api/noa-tower/respond`만 호출한다.
- 세션 상태는 브라우저가 들고 다니며, 서버는 매 턴 이를 갱신해 돌려준다.

## 7. 현재 구현 범위

- 관리자 대사 50개 구조화 완료
- 심리 벡터 휴리스틱 판정
- 특수 상황 트리거
- 층 감각/기록 상태 환산
- 사건 목표 4단계
- 단서 보드 6개
- 이론 조각 보드 6개
- 추천 다음 수 3개 자동 제안
- 최종 기록 제출 판정
- 붕괴 / 돌파 / 중단 엔딩
- 브라우저 자동 저장 및 복원
- 웹 페이지 UI와 상호작용 라우팅

## 8. 다음 확장 포인트

- 사건 팩 다중화
- 실제 퍼즐 노드와 정답 검증 연결
- 입력 문장 의미 분석 강화
- 환경 반응 대사 풀 추가
- 세이브 슬롯과 기록 재생 기능
- Hard Mode 전용 축약 대사 세트 추가
