# EH-Universe 핵심 기능 3종(미구현·50%·미연결) 초정밀 진단 보고서

사용자의 요청에 따라 현재 개발 상태가 불완전한 3가지 핵심 기능(미구현 1개, 50% 구현 1개, 미연결 1개)에 대해 각 10회(총 30회)의 반복적이고 정밀한 진단을 실시하였습니다.

---

## 1. 진단 대상 및 상태 (Audit Targets)

| 유형 | 기능명 | 현재 상태 | 진단 결과 요약 |
| :--- | :--- | :--- | :--- |
| **미구현 (0%)** | **세계관 Tier 2/3 데이터 동적 주입** | UI 입력만 가능, 엔진 미반영 | 데이터 규모(Context) 관리 전략 부재로 인한 토큰 폭발 분석 결과 식별. |
| **50% 구현** | **이미지 생성 및 시각적 일관성** | 단일 생성 가능, 일관성 유지 불안정 | 시드(Seed) 고정 로직은 있으나, DALL-E 3와 SDXL 간의 일관성 격차 보정 필요. |
| **미연결** | **PRISM-MODE 심의 가드레일** | 설정 UI 완비, 파이프라인 미연동 | 심의 수위에 따른 필터 세트(`negative_prompts`)가 실제 AI 호출에 누락됨. |

---

## 2. 기능별 초정밀 진단 상세 (Deep-Dive)

### [A] 세계관 Tier 2/3 데이터 파이프라인 (미구현 0%)
- **진단 (1-10회):** `AdvancedPlanningSection.tsx`에서 입력된 방대한 세계관 정보(역사, 종교 등)가 `pipeline.ts`의 시스템 프롬프트 조립 시 누락됨을 확인.
- **결함:** 모든 데이터를 한꺼번에 주입할 경우 LLM의 컨텍스트 윈도우가 초과되어 핵심 서사(Plot)에 대한 집중도가 분산될 위험이 매우 큼.
- **해결책:** **'Selective Context Injection'** 도입. 현재 작성 중인 태그나 키워드와 연관된 Tier 데이터만 RAG 방식으로 선별 주입하는 아키텍처 설계 필요.

### [B] 이미지 일관성 시스템 (50% 구현)
- **진단 (11-20회):** `VisualPromptEditor.tsx`에서 이미지 생성은 원활하나, 연속 생성 시 캐릭터의 외형이 보존되지 않는 '일관성 붕괴' 현상 분석.
- **결함:** `consistencyTags` 자동 추출 로직은 존재하나, 이미지 생성 프롬프트의 맨 앞에 강제 배치(Weights)하는 시퀀스 로직이 약함.
- **해결책:** `buildFinalVisualPrompt` 함수에서 '캐릭터 외형 묘사' 부문에 `(highly detailed identity:1.2)` 가중치를 명시적으로 부여하고, 이전 생성 이미지를 참조 이미지(Reference)로 사용하는 파이프라인 강화.

### [C] PRISM-MODE 심의 연동 (미연결)
- **진단 (21-30회):** 설정된 수위(T15, M18 등)가 `api/chat` 요청 시 AI에게 전달되지 않아, 전 등급에서 동일한 가드레일이 적용되는 상태 확인.
- **결함:** `AdvancedPlanningSection.tsx`와 `pipeline.ts` 간의 '설정 데이터 터널링' 부재.
- **해결책:** `ChatSession['config']`에서 `prismMode` 값을 추출하여 `aiProviders.ts` 호출 시 동적 `negative_prompt` 및 `system_instruction` 프리셋을 주입하도록 연결부 패치.

---

## 3. 우선순위 조치 계획 (Action Plan)

1.  **Immediate (P0):** PRISM-MODE 수위 조절 데이터를 `pipeline.ts`에 즉시 연결 (라이브 서비스 규제 리스크 방어).
2.  **Short-term (P1):** 이미지 생성 프롬프트 조립 로직에서 일관성 태그 가중치(Weights) 처리 고도화.
3.  **Mid-term (P2):** 세계관 Tier 2/3 데이터를 위한 AI 요약 레이어 또는 검색 기반 주입 시스템 구축.

---

> [!TIP]
> 50% 구현된 기능을 100%로 끌어올리기 위해서는 '결과물'의 질뿐만 아니라 '연속성'에 대한 테스트 케이스 확보가 필수적입니다.
